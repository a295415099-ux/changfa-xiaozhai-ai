# AI项目01 第一轮诊断报告

## 一句话结论

第一轮不建议大拆重做。建议采用：

**纯产品首页的导购逻辑 + 轻盈商详页的首屏表达 + 明星首页的信任背书模块**

先做一版更适合移动端搜索进店和快速成交的淘内承接阵地。

## 当前素材

| 素材 | 判断 |
|---|---|
| `商详页.webp` | 专业功效和技术背书很强，但首屏密度偏高，用户第一眼需要更快读懂“适合谁、解决什么、为什么买”。 |
| `轻盈商详页.webp` | 首屏更轻、更适合移动端，适合作为第一轮商详页主方向。 |
| `明星首页.webp` | 信任感和活动感强，有明星、会员权益、爆品、品牌故事，但导购路径略散。 |
| `纯产品首页.webp` | 产品入口更清楚、转化路径更直接，适合作为第一轮首页主结构。 |

## 核心问题

### 1. 首页承接：信任有了，但购买路径还不够“一眼顺”

明星首页能快速建立品牌信任，但页面同时承载明星、活动、会员、爆品、品牌故事和多 SKU 推荐，用户进来后容易被多个方向分流。

第一轮首页要解决的是：用户进店后 5 秒内知道买什么、为什么现在买、点哪里。

### 2. 商详承接：证据很多，但首屏决策还可以更短

商详页有成分、技术、显微镜、发丝对比、功效数据等强证明，但首屏需要更像“购买决策入口”：

- 一句话卖点。
- 适用人群。
- 核心功效。
- 规格/价格/活动提示。
- 立即行动入口。

### 3. 内容到页面：外部种草语言还没有被单独标记

现有页面能看出“修护、蓬松、柔顺、不毛躁、持久夹蓬”这些卖点，但还没有单独建立站外内容词、搜索词、商详词之间的连接表。

这会导致小红书、抖音或投放素材说的卖点，未必能在首页首屏、首图和商详首屏被同一句话接住。

### 4. 页面版本：已有两个方向，但还缺 A/B 判断规则

当前至少有两套可对比方向：

- 明星信任型首页。
- 产品导购型首页。
- 专业功效型商详页。
- 轻盈转化型商详页。

下一步需要把它们转成明确测试项，而不是凭感觉选。

## 第一轮优先级

| 优先级 | 动作 | 原因 |
|---:|---|---|
| P0 | 先定首页主结构：以纯产品首页为主，保留明星信任模块 | 首页先解决买什么，再解决为什么信任。 |
| P0 | 商详页首屏改成轻盈版逻辑 | 更适合手机端快速读懂和搜索承接。 |
| P1 | 建立三类词：搜索词、卖点词、场景词 | 后续接悟空、投放、社媒时都需要统一语言。 |
| P1 | 给首页和商详页增加“行动路径”检查 | 避免页面好看但不知道下一步点哪里。 |
| P2 | 上线后 7 天做数据复盘 | 判断首图点击、首页路径、商详加购和转化是否提升。 |

## 第一轮建议组合

### 首页

采用 `纯产品首页.webp` 的产品导购逻辑，叠加 `明星首页.webp` 中最强的信任资产。

首页首屏建议顺序：

1. 核心爆品/核心利益点。
2. 当前活动或会员权益。
3. 明星/信任背书。
4. 爆品分流入口。
5. 品类/场景导航。

### 商详页

采用 `轻盈商详页.webp` 的轻量首屏，保留 `商详页.webp` 中更强的专业证据模块。

商详首屏建议顺序：

1. 一句话功效承诺。
2. 适合人群。
3. 3 个核心卖点。
4. 关键证据。
5. 规格/活动/购买提示。

## 需要补的数据

| 数据 | 用途 |
|---|---|
| 首页点击路径 | 判断首页哪些模块真能导购。 |
| 商详页停留、跳失、收藏加购、转化 | 判断首屏和中段证据是否有效。 |
| 首图点击率 | 判断搜索货架第一眼是否成立。 |
| 搜索成交词 | 判断页面关键词是否接住真实搜索。 |
| 投放素材 CTR/ROI | 判断哪些素材语言应该进入首页和商详。 |

## 第一轮目标

第一轮先不追求“最完整”，而是追求“最能上线测试”：

- 首页路径更清楚。
- 商详首屏更短。
- 卖点词更统一。
- 明星信任不过度抢导购。
- 上线后 7 天能用数据判断下一轮。
