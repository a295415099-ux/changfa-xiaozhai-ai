# 商详页优化 Brief

## 项目基本信息

| 字段 | 内容 |
|---|---|
| AI 项目编号 | AI项目01 |
| 页面类型 | 商详页 |
| 阶段版本 | 01初稿 |
| 当前参考 | `assets/ecommerce/商详页.webp`、`assets/ecommerce/轻盈商详页.webp` |
| 第一轮方向 | 轻盈转化型商详页 |

## 设计目标

把商详页从“专业证明很完整”优化成“手机端能快速理解、快速信任、快速行动”的成交页面。

## 核心人群假设

| 人群 | 关注点 | 页面要接住 |
|---|---|---|
| 发质干枯毛躁人群 | 柔顺、修护、减少毛躁 | 首屏直接说清改善结果。 |
| 烫染受损人群 | 修护、强韧、减少断裂 | 中段强化技术和成分证据。 |
| 追求蓬松轻盈人群 | 不塌、不油、不厚重 | 用轻盈版视觉和体验词接住。 |
| 搜索进店新客 | 是否适合我、凭什么信 | 首屏人群、功效、证据并列出现。 |

## 首屏优化要求

首屏必须同时完成 5 件事：

1. 一句话说明产品解决什么问题。
2. 明确适合哪类头发。
3. 展示 3 个核心利益点。
4. 给出一个可信证据。
5. 保留价格/规格/活动/购买行动入口。

## 建议首屏文案结构

```text
1 抹修护枯毛躁
3 天持久柔顺亮

适合：烫染受损 / 干枯毛躁 / 蓬松易炸发
核心利益：修护、柔顺、轻盈不塌
证据：发丝修护测试 / 成分技术 / 使用前后对比
```

## 页面模块顺序

| 顺序 | 模块 | 保留/优化 |
|---:|---|---|
| 1 | 首屏卖点 | 用 `轻盈商详页.webp` 的轻量视觉方向，减少首屏拥挤感。 |
| 2 | 痛点场景 | 明确“干枯、毛躁、烫染受损、发尾打结”等场景。 |
| 3 | 使用前后对比 | 保留，但图文要更快读懂，不要只放专业数据。 |
| 4 | 技术证据 | 保留显微镜、发丝热图、修护技术，但压缩说明层级。 |
| 5 | 成分信任 | 保留 6 重植萃、油脂、护发成分。 |
| 6 | SKU/规格选择 | 增加“轻度/中度/重度受损怎么选”。 |
| 7 | 评价/问大家承接 | 后续补用户评价截图和客服高频问题。 |

## 视觉方向

- 主视觉用轻盈、顺滑、水润、发丝流动感。
- 专业证据模块可以使用黑金/显微镜风格，但不能让整页过重。
- 手机端一屏只强调一个主信息，避免多个证明同时抢注意力。
- 产品图需要在关键模块反复出现，保持记忆点。

## 成功判断

上线后重点看：

- 商详首屏跳失是否下降。
- 收藏加购是否提升。
- 详情页停留是否提升。
- 核心 SKU 转化是否提升。
- 搜索词进店后的成交是否更集中。
