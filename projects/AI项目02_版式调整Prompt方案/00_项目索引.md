# AI项目02：制定版式调整Prompt方案

## 项目定位

本项目用于把“版式调整”变成可被 AI、设计师、电商运营共同执行的 Prompt 方案体系。

它不是单纯写提示词，而是建立一套针对长发小寨品牌设计部的版式诊断、版式改稿、Prompt 输出、设计复核和版本沉淀流程。

## 覆盖范围

| 范围 | 典型对象 |
|---|---|
| 电商设计 | 商详页、店铺首页、品类页、专题页、活动页、首图、主图 |
| 品牌社媒推广 | 小红书长图、抖音封面、朋友圈海报、视频号封面、达人 brief 图 |
| 包装设计 | 包装正面、侧面信息层级、套装陈列、礼盒开箱图 |
| 汇报文件 | 单页汇报、老板版摘要、项目复盘页、SOP 图页 |
| AI 产图与改图 | 图片重排、信息层级优化、风格统一、移动端适配 |

## 与 AI项目01 的关系

AI项目01 负责淘内承接阵地，包括商详页和首页的策略、页面结构、上线检查与数据复盘。

AI项目02 负责“这些页面怎么改版式、怎么写 Prompt、怎么让 AI 或设计师按同一套标准改稿”。

简单说：

- AI项目01 判断“页面要解决什么经营问题”。
- AI项目02 输出“版式如何调整、Prompt 怎么写、交付如何检查”。

## 第一轮交付物

| 文件 | 用途 |
|---|---|
| `01_版式诊断与Prompt方案.md` | 用于判断画面问题，并输出可执行的 Prompt 改稿方案 |
| `02_版式调整Prompt模板库.md` | 沉淀商详页、首页、社媒、包装、汇报页的 Prompt 模板 |
| `03_执行检查表.md` | 给设计师、AI 操作人和负责人上线前检查使用 |

## 当前执行原则

1. 先判断画面属于哪类业务：电商、社媒、包装、汇报、AI 产图。
2. 再判断版式问题属于哪类：层级乱、重点弱、信息太满、移动端不友好、风格不统一、转化行动不清。
3. 输出 Prompt 时必须包含：目标、对象、画面比例、保留内容、调整方向、禁止项、交付格式。
4. 所有 Prompt 都要能回到品牌设计部 SOP，而不是只服务一次性改图。
5. 每次改稿都要沉淀为可复用模板。

## 后续归总规则

以后凡是出现以下任务，都可以归入 AI项目02：

- 帮我调整版式。
- 帮我写改图 Prompt。
- 这个页面信息太乱怎么改。
- 商详页 / 首页 / 海报 / 包装怎么重新排版。
- 用 AI 把这张图改高级、改清晰、改适合手机端。
- 生成一套设计师能看懂的改稿要求。
