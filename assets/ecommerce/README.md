# 电商资料库

这里放 `AI项目01：淘内承接阵地优化项目` 需要的商详页面、店铺首页、品类页、专题页、首图、活动页、投放素材和电商页面数据。

## 归档原则

电商设计资料采用“两层归档”：

1. 先按页面或物料类型归位。
2. 再在文件名里标记阶段版本。

这样既方便设计部找素材，也方便 Codex 总控判断某个页面处在初稿、打样稿、输出正稿还是上线复盘阶段。

## 页面类型目录

| 目录 | 放什么 | 典型用途 |
|---|---|---|
| `首页/` | 店铺首页、品牌首页、活动首页 | 店铺信任、核心 SKU 导流、品牌承接 |
| `品类页/` | 品类集合页、系列页、场景分类页 | 多 SKU 选择、品类教育、搜索承接 |
| `专题页/` | 节日专题、大促专题、明星专题、场景专题 | 活动转化、内容承接、短期爆发 |
| `商详页/` | 商品详情页、首屏、卖点模块、评价模块 | 单品成交、信任证明、内容到购买 |
| `首图/` | 主图、搜索首图、货架图、广告首图 | 点击率、搜索进店、投放素材 |
| `活动页/` | 618、双 11、会员日、直播承接页 | 活动路径、权益承接、转化加速 |
| `投放素材/` | 万相台、超级短视频、搜索广告素材 | 内容投放、素材复盘、优质素材筛选 |
| `页面数据/` | 点击、跳失、收藏加购、转化、ROI | 复盘判断、版本对比、经营决策 |
| `待归档/` | 临时收到但还没判断归属的资料 | 先收进来，再由 Codex 归档 |

## 阶段版本命名

文件名建议统一带上阶段：

| 阶段 | 文件名标记 | 说明 |
|---|---|---|
| 需求资料 | `00需求` | brief、截图、参考、数据 |
| 初稿 | `01初稿` | 第一版设计方向 |
| 修改稿 | `02修改稿` | 根据内部反馈调整 |
| 打样稿 | `03打样稿` | 准备测试、上架或交付确认 |
| 输出正稿 | `04正稿` | 最终交付、上线、可复用资产 |
| 上线复盘 | `05复盘` | 数据、结论、下轮动作 |

## 推荐命名格式

```text
2026-06-AI项目01-首页-01初稿-品牌承接版.webp
2026-06-AI项目01-商详页-03打样稿-轻盈款核心SKU.webp
2026-06-AI项目01-首图-04正稿-搜索主图A.webp
2026-06-AI项目01-页面数据-05复盘-7天转化.xlsx
```
