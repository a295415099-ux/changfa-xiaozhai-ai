# 外部 AI 调度原则

## 总控与外包边界

Codex 保留任务拆解、判断、调用、验证和最终交付权。外部 AI 只作为专用能力池，不替代经营判断。

悟空这类 Agent 应定位为平台专项 Agent。悟空主要服务淘宝/天猫/阿里生态，适合被 Codex 调度处理淘内搜索、商品承接、店铺优化、投放和成交复盘；不应作为长发小寨跨平台品牌设计与经营总控。

| 能力 | 默认工具 | 使用边界 |
|---|---|---|
| 经营诊断与架构 | Codex | 负责主判断、Skill 路由、方案合并、交付物结构 |
| 中文语义与商品识别 | 千问/豆包/通义 | 用于商品图识别、卖点扩写、问答语料、初稿生成 |
| 电商生图 | 稿定AI、OpenAI 图像、Gemini、Liblib | 用于详情图、场景图、产品图、海报方向 |
| 视频生成 | 即梦、可灵、万相、Seedance/Veo 类模型 | 用于产品展示视频、口播视频、短视频素材 |
| 阿里生态数据 | 淘宝/天猫、生意参谋、万相台等导出表 | 用于搜索、投放、承接、成交、复盘判断 |
| 淘宝/天猫专项 Agent | 悟空等阿里生态 Agent | 用于淘内搜索、商品页、店铺、投放、成交建议，由 Codex 统一验收 |
| 项目自动化 | Codex | 用于模板处理、表格生成、脚本、内部知识库和 Skill 管理 |

## 标准调度步骤

1. Codex 先明确任务和输出物。
2. 需要外部 AI 时，只给它最小必要输入。
3. 外部 AI 返回后，Codex 按经营链路检查：语义是否一致、关键词是否统一、承接是否成立、指标是否可复盘。
4. 最终结果由 Codex 汇总成老板版、团队版、复盘版。

## 悟空式能力可吸收点

- 多 Agent 分工：一个总控负责拆任务，多个专项 Agent 负责文档、内容、数据、会议、执行。
- 企业工具连接：把聊天、文档、表格、会议、任务管理、投放数据连接起来。
- 权限与审计：AI 可以建议和生成，但关键拍板、预算、上架、投放需要人确认。
- 过程沉淀：每次诊断、复盘、脚本、关键词、素材评分都要回写为品牌资产。

## 平台 Agent 接入规则

每接入一个新 Agent，先登记四项信息：

1. 平台/业务域：淘宝、小红书、抖音、微信、包装供应链、数据 BI 等。
2. 能力边界：能读什么、能生成什么、能不能执行动作。
3. 验收标准：品牌一致性、链路完整性、数据可验证性、合规性。
4. 沉淀位置：品牌资产库、内容资产库、设计资产库、数据复盘库。

任何平台 Agent 的输出都必须回到 Codex 做跨平台校验。

## 长发小寨优先接入顺序

1. Jessie Skill 库：先变成稳定的内部 SOP 能力。
2. 品牌知识库：品牌定位、SKU、卖点、关键词、内容资产、店铺承接页。
3. 表格模板：复盘模板、投放模板、甘特图、行动清单。
4. 外部生成工具：生图、视频、口播、详情页。
5. 阿里生态数据与悟空：搜索词、投放、成交、承接数据导出或由悟空分析后，再由 Codex 做总判断。
6. 其他平台 Agent：小红书、抖音、微信私域、包装供应链、BI 数据工具逐步接入。
