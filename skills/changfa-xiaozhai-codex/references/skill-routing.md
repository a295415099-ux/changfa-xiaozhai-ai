# Jessie Skill 路由表

## 总体模块

| 模块 | 经营问题 | 主 Skill | 辅助 Skill |
|---|---|---|---|
| AI语义资产 | 让品牌/商品被 AI 理解、引用、推荐 | `brand-semantic-asset-sop` | `ai-product-semantic-reconstruction`, `ai-content-reference-map`, `ai-recommend-link-check`, `ai-trust-evidence-chain` |
| 诊断复盘 | 找出品牌内容流量卡点和大促后可复用资产 | `brand-content-flow-diagnosis` | `post-promo-content-review`, `ai-brand-content-blockage`, `ai-business-judgment` |
| 洞察规划 | 从竞品、同行、趋势、人群中找机会 | `competitor-trend-insight` | `high-value-audience-breakdown`, `ai-trend-track-judgment`, `ai-content-opportunity`, `ai-content-traffic-map` |

## 场景快捷路由

### 新品牌或冷启动

使用顺序：`brand-semantic-asset-sop` -> `brand-content-flow-diagnosis` -> `high-value-audience-breakdown` -> `ai-content-traffic-map` -> `win-content-design` -> `ai-content-standard-template` -> `ai-back-to-store-7day-sop`

输出：品牌语义资产、内容流量地图、首批内容方向、7天动作。

### 成熟品牌做下半年增长

使用顺序：`post-promo-content-review` -> `competitor-trend-insight` -> `ai-content-opportunity` -> `keyword-unified-sop` -> `u-shape-content-reflow` -> `ai-60day-training-camp`

输出：可复用内容资产、机会池、关键词资产、回流路径、60天推进表。

### 内容有流量但没成交

使用顺序：`ai-content-chain-check` -> `ai-content-search-connect` -> `ai-external-content-internal-receiving` -> `ai-store-reception-optimization`

输出：断链检查、内容搜索承接连接表、外种内收路径、店铺承接优化动作。

### 站外种草不回淘

使用顺序：`keyword-unified-sop` -> `u-shape-content-reflow` -> `ai-search-reflow-observation-sop` -> `ai-content-collaboration`

输出：三类词表、U型回流路径、搜索回流观察表、内容投放协同表。

### 大促前备战

使用顺序：`ai-trend-track-judgment` -> `ai-content-water-discrimination` -> `quality-material-screening` -> `ai-content-reception-checklist` -> `ai-post-training-gantt-chart`

输出：趋势判断、蓄水判断、优质素材清单、承接检查、甘特图。

### 团队执行混乱

使用顺序：`ai-roles-division` -> `team-leader-promotion-sop` -> `ai-back-to-store-7day-sop` -> `ai-60day-training-camp`

输出：三角色分工、总推进人机制、7天行动、30/60天陪跑任务。

## 路由判断规则

- 用户问「怎么看」「卡在哪」「为什么没转化」：先诊断类。
- 用户问「做什么方向」「机会在哪」：先洞察规划类。
- 用户问「帮我写」「生成脚本/笔记」：先检查关键词与承接，再进入内容生产类。
- 用户问「外面种草怎么回淘」「搜索怎么做」：先搜索回流类。
- 用户问「详情页/店铺怎么接」：先承接转化类。
- 用户问「谁来做」「怎么推进」：先执行复盘类。
- 用户问「AI怎么推荐我们」「品牌怎么被AI理解」：先 AI 语义资产类。
