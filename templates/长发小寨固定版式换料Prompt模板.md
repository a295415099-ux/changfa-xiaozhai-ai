# 长发小寨固定版式换料 Prompt 模板

## 文档定位

这份模板是团队内部用于“固定版式，只换部分内容”的 AI 作图标准。适用于电商主图、达人专属图、机制图、产品组合图、指定背景主图、透明 PNG 产品素材、高清修复等任务。

更新版本：v2  
更新时间：2026-06-30  
对应总控项目：AI项目02：制定版式调整Prompt方案

母 Prompt 入口：

[打开长发小寨固定版式换料 Prompt 模板 v1](https://chatgpt.com/g/g-6a437898900481918e3f6fd9a1c281a0-chang-fa-xiao-zhai-gu-ding-ban-shi-huan-liao-prompt-mo-ban-v1)

使用原则：

- 保版式：构图、边框、品牌风格、Logo 位置和主体排版尽量不变。
- 换素材：只替换明确指定的人物、产品、文案、颜色、背景、赠品、数量、标签。
- 换背景：如果指定更换背景，要保留原图主体排版逻辑，只让新背景服务产品和文案，不重新设计整张图。
- 控文字：中文必须单独检查，避免多字、少字、错字、变形字。
- 分轮返修：复杂任务先换主体，再修文字和细节。
- 团队交付：优先沉淀为团队可共享的 GPT、Figma、Canva、PSD 或 Adobe Cloud 模板，不做只存在个人本机的文件。

## 与 AI项目02 的关系

这份模板是 AI项目02 的一个执行分支，负责“固定版式换料”。

| 层级 | 作用 |
|---|---|
| AI项目02：制定版式调整Prompt方案 | 判断画面问题、输出版式调整 Prompt、沉淀团队模板 |
| 本模板：固定版式换料 Prompt | 在既有版式不变的情况下，换产品、换文案、换背景、换达人、换机制 |
| AI项目01：天猫首页及发膜0627设计初稿 | 决定商详页、首页、主图等页面到底要解决什么经营问题 |

简单判断：

- 如果“整体版式也要优化”，先走 AI项目02 的版式诊断。
- 如果“版式已经确定，只换内容”，直接用本模板。
- 如果“换完后要上线测试”，再回到 AI项目01 做数据复盘。

```text
本次具体修改：
{把具体需求写在这里}
```

```text

1. 品牌 Logo 位置和大小关系。

本次只替换：
1. {要替换的产品/人物/文案/背景/颜色/赠品/标签/数量}

版式要求：
1. 主信息必须第一眼可读，手机端缩小后仍然清楚。
2. 产品或人物不能被文字遮挡，主体边缘要干净。
3. 新元素的透视、比例、光影和遮挡关系要自然融入原图。

3. 不要新增未经确认的数据、医疗化表达、极限词或虚假背书。
```

## 单项修改模板

### 只改文案

```text
请在保持原图版式、背景、产品、人物和品牌 Logo 不变的前提下，只修改画面中的文字。

把原文字：
「{原文案}」

改成：
「{新文案}」

要求：
1. 中文必须完全正确。
2. 字体风格、字号层级、颜色、阴影、排版位置尽量延续原图。
3. 不要改动产品、人物、背景和边框。
4. 如果新文案长度不同，请在原文字区域内合理排版，不要遮挡产品和人物。
```

### 只换产品

```text
请基于参考图，只替换画面中的产品，其他版式不变。

替换要求：
1. 将原来的产品替换为：{产品名称/数量/规格}。
2. 产品摆放位置参考原图，保持同样的透视、大小关系和视觉重心。
3. 如果是多件产品，请按照电商主图方式自然排列，前后层次清楚。
4. 保留背景、人物、Logo、标题文字、边框和整体色调。
5. 产品标签和瓶身/盒身尽量清晰真实，不要生成错误品牌或乱码文字。
```

### 只换人物或达人

```text
请基于参考图，只替换画面中的人物形象，其他元素保持不变。

人物替换要求：
1. 使用我上传的人物照片作为参考。
2. 保持人物在原图中的位置、大小、姿势风格和商业海报质感。
3. 人物边缘要干净，自然融入背景。
4. 保留人物下方专属标签区域。
5. 标签文字改为：「{达人姓名}专属」。
6. 不要改变产品、背景、主标题、品牌 Logo 和边框。
```

### 改赠品或下单机制

```text
请基于参考图，只修改赠品区域/机制区域。

修改内容：
1. 将「赠品」改为「下单即得」。
2. 将「体验装」改为「旅行装」。
3. 保留原有红色/卡片式区域、图标、边框和排版风格。
4. 赠品小图数量和摆放方式保持清晰，不要遮挡主体产品。
5. 其他区域不要改动。
```

### 只改价格、权益或活动机制

```text
请基于参考图，只修改价格、权益或活动机制区域，其他区域保持不变。

修改内容：
1. 原价格/权益/机制为：「{原内容}」。
2. 修改为：「{新内容}」。
3. 如果涉及券、赠品、到手价、限时活动，请保持原来的卡片结构和视觉层级。
4. 数字、单位、符号必须准确，例如 ¥、元、件、瓶、g、mL。
5. 不要改动产品、人物、背景、主标题、品牌 Logo 和边框。

复核重点：
1. 价格是否与运营确认一致。
2. 活动机制是否完整。
3. 是否有多字、少字、错字。
4. 是否误导用户或出现夸大承诺。
```

## 手机端 / PC 端适配检查

固定版式换料完成后，必须按使用场景检查一次。

| 使用场景 | 检查重点 |
|---|---|
| 淘宝手机端主图 | 缩小后主标题是否可读，产品是否清楚，利益点是否一眼能看懂 |
| 淘宝商品卡 | 缩略图中产品轮廓是否突出，卖点是否过多，背景是否太花 |
| 商详页模块 | 一屏是否只讲一个重点，文字是否过密，证据是否能读懂 |
| 店铺首页 | 主推 SKU 是否比普通模块更突出，点击路径是否清楚 |
| 小红书图 | 标题是否有停留理由，画面是否像内容而不是硬广 |
| 抖音封面 | 文字是否足够大，人物/产品是否有强视觉焦点 |
| 包装图 | 品牌名、产品名、功效、系列区分是否远看能识别 |

## 合规安全检查

所有固定版式换料，都要避免因为“换一句文案”带来合规风险。

| 检查项 | 要求 |
|---|---|
| 功效宣称 | 不新增未经确认的防脱、生发、治疗、医学级承诺 |
| 数据表达 | 不编造百分比、销量、榜单、检测结果 |
| 明星达人 | 不使用未经授权的明星、达人肖像或背书 |
| 价格机制 | 到手价、赠品、活动时间必须与运营确认 |
| 中文文字 | 人工复核，不完全依赖 AI |
| 产品包装 | 不改变真实包装结构、品牌名和核心标签 |

## 能力边界

### 完全可以做

- 固定版式下替换产品组合、人物照片、主题文案、底部文案、赠品机制。
- 使用指定背景做主图，或把山水、促销、纯色、节日等背景替换到同一套版式中。
- 按已有图风格扩展同系列主图、达人专属图、机制图。
- 把单个产品处理成透明 PNG。
- 做高清化、去噪、锐化、背景清理。
- 做方图、竖图、主图、贴片、机制图等不同用途版本。

### 需要复核

- 中文文字是否一次性完全正确。
- 产品标签小字、品牌 Logo、瓶身细节是否完全还原。
- 多产品的透视、光影、遮挡关系是否足够自然。
- 指定背景与产品光影是否完全匹配。
- 低清截图能否恢复成真正高清细节。
- 透明 PNG 的边缘是否有毛边。

### 不建议只靠 Prompt 执行

- 像 PSD/Figma 一样要求像素级精准排版。
- 要求所有中文、小字、产品标签一次性 100% 正确。
- 凭空生成完全真实、可直接商用的新包装细节。
- 伪造未经授权的明星、达人、品牌背书。
- 生成虚假功效、医疗级承诺、夸大宣传图。
- 从模糊截图恢复出原始高清包装细节。

## PSD / Figma / Canva / Adobe 协作原则

如果任务需要长期给团队复用，不建议只把最终 JPG/PNG 发给团队，应该尽量沉淀成可编辑模板。

### 推荐交付形态

- Figma：适合做团队共享版式模板、组件化主图、多人协作审核。
- Canva：适合运营团队快速替换文案、产品图、背景和活动机制。
- PSD：适合设计师保留精修图层、光影、蒙版、产品合成细节。
- Adobe Cloud / Adobe ID：适合团队共享 PSD、素材库、品牌资产和版本记录。
- GPT 母 Prompt：适合非设计同事快速提交换料需求，统一表达口径。

### 团队交付最低标准

- 模板必须放在团队可访问的位置，不只存在个人电脑。
- 必须保留可编辑源文件：Figma 链接、Canva 模板链接、PSD 云文件或团队网盘源文件。
- 图层命名要让团队看得懂，例如：背景、主标题、产品组合、赠品区、达人头像、品牌 Logo、阴影、光效。
- 文案区、产品区、背景区要尽量分层，方便后续单独替换。
- 每个模板要附一条“适用场景”，例如：补贴福利主图、达人专属主图、产品组合机制图、透明 PNG 产品素材。

### 不建议的交付方式

- 只交一张合成 PNG，没有源文件。
- 只保存在个人 Canva、个人 Figma、个人 Adobe ID，团队无法编辑。
- PSD 图层全部合并，后续无法换背景、换产品、换文案。
- 模板没有命名规则，团队无法判断哪个是最新版。

## 背景替换任务调用细化

当团队收到类似“帮忙用这个背景做一下主图，产品组合是 139：控油防脱 400g + 固发防脱 400g + 控油防脱 50g + 固发防脱 50g + 养发乳 50g”这种需求时，按下面结构调用。

### 第一步：先判断任务类型

```text
任务类型：指定背景主图 / 固定版式换背景 / 产品组合机制图
参考背景：已上传红色补贴福利背景
产品组合：控油防脱400g + 固发防脱400g + 控油防脱50g + 固发防脱50g + 养发乳50g
主文案：补贴福利 先到先得
输出用途：电商主图
输出比例：沿用参考图比例，或按平台要求输出 1:1
```

### 第二步：复制给 GPT 的完整 Prompt

```text
请基于我上传的背景图制作一张电商主图。

这次任务不是重新设计整套视觉，而是使用指定背景完成产品组合主图。

固定要求：
1. 使用我上传的红色补贴福利背景作为主背景。
2. 保留背景中的促销氛围和主文案感觉。
3. 画面整体要像电商主图，清晰、干净、促销感强。
4. 产品必须是画面主体，背景不能抢产品。
5. 产品、文字、阴影、地台和光效要自然融合。

产品组合：
控油防脱400g + 固发防脱400g + 控油防脱50g + 固发防脱50g + 养发乳50g

文案要求：
主文案保留或改为：「补贴福利 先到先得」
不要生成多余文字，不要生成乱码，不要添加无关水印。

排版要求：
1. 主标题放在画面上方或背景原有标题区域。
2. 产品组合放在画面中下部，作为视觉中心。
3. 小规格产品可以放在前排，大规格产品放在后排。
4. 产品数量必须准确，不要多放、少放。
5. 保持产品包装真实，不要改变瓶身、管身、标签结构。

输出要求：
1. 输出电商主图效果。
2. 画面清晰，产品边缘干净。
3. 如果文字不确定，请优先保证产品和版式正确，文字后续单独返修。
```

### 第三步：如果要沉淀成团队模板

```text
请在生成主图的同时，按团队可复用模板逻辑输出分层建议：

1. 背景层：红色补贴福利背景。
2. 主标题层：补贴福利 先到先得。
3. 产品组合层：控油防脱400g、固发防脱400g、控油防脱50g、固发防脱50g、养发乳50g。
4. 阴影/地台层：承接产品，不遮挡文字。
5. 可替换区域：背景、主文案、产品组合、小规格赠品。

如果后续进入 Figma / Canva / PSD，请按这些图层命名，方便团队继续换背景、换产品、换文案。
```

### 第四步：返修调用

```text
请只修正以下问题，其他正确区域保持不变：

1. 产品组合必须是：控油防脱400g + 固发防脱400g + 控油防脱50g + 固发防脱50g + 养发乳50g。
2. 主文案必须是：「补贴福利 先到先得」。
3. 背景继续使用红色补贴福利背景，不要换成其他背景。
4. 产品不要变形，数量不要增加或减少。
5. 不要重新设计整张图，只修正错误区域。
```

### 第五步：团队交付判断

| 交付需求 | 推荐方式 |
|---|---|
| 只要一张快速主图 | GPT 出图 + 人工检查 |
| 以后经常换产品 | Canva 或 Figma 模板 |
| 需要设计师精修光影 | PSD 分层文件 |
| 需要多人长期共用 | Figma / Canva 团队链接 |
| 需要沉淀为 AI 标准流程 | GPT 母 Prompt + 总控台文档 |

## 团队执行流程

1. 先上传参考母版图。
2. 上传需要替换的人物、产品、Logo、背景图或透明 PNG。
3. 使用“通用母 Prompt”说明固定保留内容。
4. 把本次修改逐条编号，复杂任务不要混在一句话里。
5. 第一轮只看主体是否对：人物、产品、数量、版式。
6. 第二轮再修文字、边缘、光影、细节。
7. 如果涉及背景替换，单独检查背景是否干扰产品和文案。
8. 最终交付前由人工检查中文、产品数量、功效宣称和品牌合规。
9. 需要团队长期复用的图，必须同步沉淀为 Figma、Canva、PSD 或 Adobe Cloud 模板链接。

## 总控台归档规则

每次固定版式换料完成后，建议同步归档到长发小寨 AI 总控台，便于后续复用。

| 字段 | 填写内容 |
|---|---|
| 所属项目 | AI项目02：制定版式调整Prompt方案 |
