# AI商品语义重构表 SOP

## 核心目标

解决传统商品表达只会写材质、颜值、销量、功能、性价比，无法让AI理解这款商品**适合谁、适合什么场景、解决什么问题、为什么比同类更适合**的问题。

**核心理念：AI推荐的不是单个商品，而是一个更适合用户需求的解决方案。**

## 流程概览

共21个步骤（步骤0-20），分5个阶段：

| 阶段 | 步骤范围 | 核心任务 |
|------|----------|----------|
| 基础准备 | 0-2 | 任务定义、资料回收、商品身份确认 |
| 语义盘点 | 3-9 | 人群、场景、痛点、卖点、情绪、证据、对比优势 |
| 语义生成 | 10-14 | 购买理由、场景标签、问答语料、推荐话术、推荐卡 |
| 质量检查 | 15-16 | 语义完整度检查、AI可读性检查 |
| 落地应用 | 17-20 | 重构表整合、缺口输出、团队分工、滚动更新 |

## 执行原则

1. **必须按步骤顺序执行**，不可跳步。前一步的输出是后一步的输入。
2. **每一步都生成明确的表格/文档输出**，不可只口头讨论。
3. **所有输出使用用户语言而非品牌语言**，确保AI能理解关系和理由。
4. **避免泛化表达**：不写"所有人都适合"，不写"更好"，要写具体人群、具体优势。

## 阶段一：基础准备（步骤 0-2）

### 步骤 0：明确本次任务

**任务目标**：统一认知——这一步不是补几条卖点，而是重构商品语义。

**具体动作**：
- 向用户确认：本次要处理的商品名称、现有商品资料（详情页、卖点稿等）
- 明确最终交付物：《AI商品语义重构表》《单品AI推荐理由卡》《商品场景标签表》《商品问答语料库》
- 强调核心理念：传统商品写参数，AI友好商品要写适配关系与解决方案

**输出**：商品语义重构任务定义（确认商品、确认资料、确认交付物范围）

### 步骤 1：回收商品基础资料

**任务目标**：先把商品底层资料拉出来。

**具体动作**：
- 请用户提供：商品详情页链接/截图、商品卖点稿、用户评价、客服高频问题记录
- 汇总：商品名称、规格、价格带、主要功能、主要卖点
- 如果用户无法提供客服/评价数据，则根据商品信息推演典型用户关注点

**输出**：商品基础信息表（至少包含名称、规格、价格、功能、卖点5个维度）

**注意事项**：不先看清商品底层信息，后面重构容易空泛。

### 步骤 2：确认商品是什么

**任务目标**：明确这款商品的基础身份（不是型号，而是用户价值身份）。

**具体动作**：
- 用一句话说清：这款商品是什么、属于什么类型、面向什么需求
- 确认商品的"用户价值身份"：比如"这不是一款保温杯，而是一个让你随时随地喝到热水的移动饮水解决方案"

**输出**：商品身份表

## 阶段二：语义盘点（步骤 3-9）

### 步骤 3：盘点适合人群

**任务目标**：明确商品最适合哪类用户购买。

**具体动作**：
- 按核心人群分类整理：谁最适合、谁最容易转化、谁最有长期价值
- 每个群体写清特征标签（年龄/职业/生活方式/消费习惯/需求特征）
- 来源参考：用户画像、成交人群特征、客服咨询内容、评论区反馈

**输出格式参考**：
```
适合人群表：
| 人群分类 | 特征标签 | 核心需求 | 转化理由 |
|---------|---------|---------|---------|
```

**注意事项**：不能泛写"所有人都适合"，要明确圈层和标签。

### 步骤 4：盘点使用场景

**任务目标**：明确商品最适合在哪些场景出现。

**具体动作**：
- 提炼商品的典型使用场景：生活/工作/消费/送礼/家庭/出行等
- 每个场景写明：什么时间、什么地点、什么状态、和谁一起
- 来源参考：使用情境、种草内容、用户反馈

**输出格式参考**：
```
使用场景表：
| 场景分类 | 具体情境 | 用户状态 | 商品角色 |
|---------|---------|---------|---------|
```

**注意事项**：没有场景，商品很难变成解决方案。

### 步骤 5：盘点核心痛点

**任务目标**：明确用户为什么需要它。

**具体动作**：
- 把用户购买前最担心、最想解决、最容易被打动的问题整理出来
- 用用户语言表达痛点（不是"产品XX性能不足"，而是"每次XXX都让人烦躁"）
- 来源参考：评论区吐槽、客服高频问题、竞品差评、用户调研

**输出格式参考**：
```
核心痛点表：
| 痛点描述(用户语言) | 严重程度 | 当前替代方案 | 本商品如何解决 |
|-------------------|---------|------------|--------------|
```

**注意事项**：痛点要用用户语言表达，而不是品牌语言。

### 步骤 6：盘点功能卖点

**任务目标**：明确商品具体解决什么问题。

**具体动作**：
- 把商品的功能、体验、设计、服务优势按"问题解决逻辑"重写
- 每个卖点的结构：「用户遇到X问题 → 本商品通过Y功能解决 → 带来的改变是Z」
- 来源参考：商品详情页、功能说明、卖点稿

**输出格式参考**：
```
功能卖点表：
| 用户问题 | 对应功能 | 解决原理 | 用户感知收益 |
|---------|---------|---------|------------|
```

**注意事项**：卖点不能只写"好"，要写解决什么问题。

### 步骤 7：盘点情绪价值

**任务目标**：明确商品除了功能之外还带来什么情绪收益。

**具体动作**：
- 提炼生活感、身份感、审美感、陪伴感、安心感、成就感等情绪价值
- 用用户感受语言表达：拥有后的心理状态、使用时的情绪体验
- 来源参考：品牌调性、用户反馈、内容素材、评论中的情绪表达

**输出格式参考**：
```
情绪价值表：
| 情绪类型 | 触发情境 | 用户感受描述 | 证据(评论/素材) |
|---------|---------|------------|--------------|
```

**注意事项**：AI推荐越来越看重情绪价值，不只是参数价值。

### 步骤 8：盘点信任证据

**任务目标**：明确有哪些真实支撑证据。

**具体动作**：
- 整理：用户评价精选、使用案例、销量数据、达人推荐内容、资质认证、历史积累、售后服务承诺
- 每项证据标注：来源、可信度、对购买决策的影响强度
- 来源参考：评价截图、案例、销量数据、达人内容、资质材料

**输出格式参考**：
```
信任证据表：
| 证据类型 | 具体内容 | 来源 | 可信度 | 影响力 |
|---------|---------|-----|-------|-------|
```

**注意事项**：没有证据，推荐理由不够可信。

### 步骤 9：盘点对比优势

**任务目标**：明确和同类商品相比为什么选它。

**具体动作**：
- 提炼商品与竞品的差异点、替代理由、适配优势
- 不泛写"更好"，而是写"更适合什么需求"
- 来源参考：竞品资料、差异点、用户对比反馈

**输出格式参考**：
```
对比优势表：
| 对比维度 | 同类常见情况 | 本商品差异点 | 谁更适合本商品 |
|---------|------------|------------|--------------|
```

**注意事项**：对比优势要写明"更适合谁的需求"，不是贬低竞品。

## 阶段三：语义生成（步骤 10-14）

### 步骤 10：生成购买理由

**任务目标**：把卖点升级成AI可调用的购买理由。

**具体动作**：
- 整合前面所有模块（人群、场景、问题、卖点、证据）
- 按以下逻辑生成购买理由：
  1. 人群定位式：「如果你/你是X人群，正在为Y问题困扰」
  2. 场景切入式：「当你处于X场景时，你需要Y解决方案」
  3. 问题解决式：「X问题最好的解决方式是Y，因为Z」
- 每条购买理由要可直接用于推荐，不只是内部总结

**输出格式参考**：
```
购买理由表：
| 理由编号 | 适配人群 | 适用场景 | 核心购买理由 | 证据支撑 |
|---------|---------|---------|------------|---------|
```

### 步骤 11：生成场景标签

**任务目标**：把场景信息结构化为可检索标签。

**具体动作**：
- 基于步骤4的场景表和步骤5的痛点表
- 生成三个维度的标签体系：
  - 场景标签：「办公场景」「户外出行」「睡前放松」
  - 需求标签：「快速解渴」「送礼体面」「缓解疲劳」
  - 搭配标签：「搭配XX使用」「替代XX」「升级XX」

**输出格式参考**：
```
商品场景标签表：
| 标签类型 | 标签名称 | 关联场景 | 检索关键词 | 适用推荐逻辑 |
|---------|---------|---------|-----------|------------|
```

**注意事项**：这是后续AI推荐和内容检索的重要底层结构。

### 步骤 12：生成商品问答语料

**任务目标**：让商品具备可对话能力。

**具体动作**：
- 围绕用户常问问题生成标准问答语料
- 问题来源：客服高频问题、评论区常见疑问、竞品评论区问题
- 每个问答包含：问题(用户语言)、标准回答(AI可读、结构化)、追问处理

**输出格式参考**：
```
商品问答语料库：
| 问题分类 | 用户问法 | 标准回答 | 可追问方向 |
|---------|---------|---------|----------|
```

**注意事项**：问答语料要真实、具体、可复用。

### 步骤 13：生成AI推荐话术

**任务目标**：让AI在特定需求下会推荐这款商品。

**具体动作**：
- 当用户提出某类需求时，生成适合的推荐句式和推荐逻辑
- 话术结构：识别需求 → 匹配商品 → 说明理由 → 提供证据
- 针对不同人群/场景生成不同版本

**输出格式参考**：
```
AI推荐话术库：
| 触发条件 | 推荐话术模板 | 推荐逻辑 | 适用渠道 |
|---------|------------|---------|---------|
```

**注意事项**：推荐话术不是硬广，而是基于用户需求的匹配逻辑。

### 步骤 14：生成单品AI推荐理由卡

**任务目标**：把整套信息压缩成一张可调用卡片。

**具体动作**：
- 将商品身份、人群、场景、问题、卖点、证据、推荐话术收口
- 生成结构化的单品推荐卡，每个字段都是AI可调用的

**输出格式参考**：
```
单品AI推荐理由卡：
商品名称：
商品身份（一句话）：
适合人群（3类）：
核心场景（3-5个）：
核心痛点（3个）：
核心卖点（3个）：
情绪价值（2-3个）：
信任证据（3-5条）：
对比优势（3条）：
一句话推荐理由：
AI推荐触发词：
```

**注意事项**：这是最适合现场展示和后续调用的成果之一。

## 阶段四：质量检查（步骤 15-16）

### 步骤 15：检查商品语义是否完整

**任务目标**：判断商品是否已从参数表达升级为解决方案表达。

**具体动作**：
- 逐项检查是否已覆盖：人群、场景、问题、卖点、证据、搭配、禁忌、售后
- 输出完整度评分和缺失项清单

**输出格式参考**：
```
商品语义完整度检查表：
| 检查项 | 是否已覆盖 | 完整度(1-5) | 缺失说明 |
|-------|----------|------------|---------|
| 适合人群 | ✅/❌ | | |
| 使用场景 | ✅/❌ | | |
| 核心痛点 | ✅/❌ | | |
| 功能卖点 | ✅/❌ | | |
| 情绪价值 | ✅/❌ | | |
| 信任证据 | ✅/❌ | | |
| 对比优势 | ✅/❌ | | |
| 搭配建议 | ✅/❌ | | |
| 使用禁忌 | ✅/❌ | | |
| 售后服务 | ✅/❌ | | |
```

**注意事项**：缺少其中一层，AI理解就不完整。

### 步骤 16：检查商品语义是否AI可读

**任务目标**：判断这些表达是否适合AI理解与调用。

**具体动作**：
- 筛掉过于参数化、口号化、宣传化的表达
- 保留结构清楚、关系明确、可调用的表达
- 检查标准：
  - ✅ 有明确的人群→商品匹配关系
  - ✅ 有场景→需求→解决方案链条
  - ✅ 标签化、可检索
  - ❌ 纯参数堆砌（如"300ml容量、316不锈钢"）
  - ❌ 空洞口号（如"品质之选"、"匠心之作"）

**输出**：AI可读性检查表（标注需要修改的内容和建议修改方向）

## 阶段五：落地应用（步骤 17-20）

### 步骤 17：形成《AI商品语义重构表》初版

**任务目标**：把所有结果沉淀成正式工具表。

**具体动作**：
- 将商品名称、人群、场景、痛点、卖点、情绪价值、证据、对比优势、推荐话术整合成正式版
- 格式统一、字段完整、可直接使用

**输出**：AI商品语义重构表初版（完整整合）

**注意事项**：这是 SKILL 2 的核心正式输出。

### 步骤 18：输出当前最关键的商品语义缺口

**任务目标**：帮团队知道先补什么。

**具体动作**：
- 从完整度检查表（步骤15）中找出最需要补齐的 1-3 个模块
- 按优先级排序：对AI推荐影响最大的优先补
- 给出每个缺口的补齐建议和数据来源建议

**输出**：当前商品语义缺口清单（优先级排序 + 补齐建议）

**注意事项**：先补最影响AI推荐的部分。

### 步骤 19：分配到团队使用

**任务目标**：让重构结果进入内容、电商、客服、投放等使用场景。

**具体动作**：
- 明确每个产出物的使用团队和使用方式：
  - 推荐理由 → 运营/推广团队 → 详情页、广告文案
  - 场景标签 → 内容团队 → 种草内容、短视频选题
  - 问答语料 → 客服团队 → 培训素材、自动回复
  - 推荐话术 → AI运营团队 → AI客服、AI推荐系统

**输出**：商品语义使用分工表

**注意事项**：不进入团队使用，就只是一次盘点。

### 步骤 20：建立滚动更新机制

**任务目标**：让商品语义随着商品与市场变化更新。

**具体动作**：
- 设计更新规则：新品上市时、大促活动时、用户反馈集中时、竞品变化时
- 明确更新频率和责任人
- 建议更新触发条件和操作流程

**输出**：商品语义更新机制表

**注意事项**：商品语义库必须动态维护，不是一次性文档。
