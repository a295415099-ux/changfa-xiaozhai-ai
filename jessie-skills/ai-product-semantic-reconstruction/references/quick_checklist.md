# 快速检查清单

执行AI商品语义重构过程中，每完成一个阶段后逐项自检。

## 阶段一：基础准备检查

- [ ] 商品名称已确认
- [ ] 商品详情页/卖点稿资料已收集
- [ ] 用户评价/客服记录已获取（或已推演）
- [ ] 商品身份一句话定义已完成（不是型号，是用户价值身份）
- [ ] 商品基础信息表已生成（名称/规格/价格/功能/卖点）

## 阶段二：语义盘点检查

- [ ] 适合人群：已分类3-5个核心人群，不泛写"所有人"
- [ ] 使用场景：已覆盖3个以上典型场景，每个场景有时间/地点/状态
- [ ] 核心痛点：已用用户语言描述（非品牌语言），至少3个
- [ ] 功能卖点：每个卖点都包含「问题→功能→改变」逻辑链
- [ ] 情绪价值：已覆盖功能之外的感受层（生活感/身份感/安心感等）
- [ ] 信任证据：至少3种证据类型（评价/案例/销量/达人/资质等）
- [ ] 对比优势：已写清"更适合什么需求"（而非泛写"更好"）

## 阶段三：语义生成检查

- [ ] 购买理由：每条都包含人群定位+场景+理由+证据
- [ ] 场景标签：已生成场景标签/需求标签/搭配标签三类
- [ ] 问答语料：每个问答有用户问法+标准回答+可追问方向
- [ ] 推荐话术：话术模板包含「识别需求→匹配商品→说明理由→提供证据」
- [ ] 推荐理由卡：一页式汇总，所有字段已填充

## 阶段四：质量检查

- [ ] 完整度检查：10个检查项全部过检，缺失项已标注
- [ ] AI可读性检查：已筛掉纯参数/空洞口号，保留结构化表达
- [ ] 关键缺口已识别（1-3个）

## 阶段五：落地应用检查

- [ ] 重构表初版已整合所有模块
- [ ] 语义缺口清单已排序（最影响AI推荐的优先）
- [ ] 使用分工表已明确（推荐理由→谁用、标签→谁用、语料→谁用、话术→谁用）
- [ ] 更新机制已设定（触发条件+频率+责任人）
