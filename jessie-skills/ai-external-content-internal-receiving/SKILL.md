# AI外种内收链路图 SOP

## 核心理念

**外种只是第一步，内收才决定最后的结果。**

不是站外没价值，而是站内没接住；不是内容没热度，而是流量没有成交闭环。

## 标准链路定义

外种内收是一条完整的五段链路：

```
站外触点 → 种下的内容心智 → 回淘动作 → 站内承接点 → 最终转化动作
```

| 链路环节 | 回答的问题 | 具体含义 |
|---------|-----------|---------|
| 站外触点 | 在外面通过什么影响用户 | 小红书、抖音短视频、视频号、达人内容、社群内容等 |
| 种下的内容心智 | 内容到底在种什么 | 搜索词、卖点、场景、品牌印象、活动预期 |
| 回淘动作 | 用户被种草后下一步做什么 | 去搜、进商品页、进店、看直播、关注活动页 |
| 站内承接点 | 流量回来后由什么页面接住 | 商品页、店铺页、活动页、直播间、站内内容模块 |
| 最终转化动作 | 最终导向什么结果 | 收藏加购、下单、直播转化、会员沉淀、私域转化 |

## 执行流程（17步SOP）

按以下阶段和步骤依次推进。每完成一步，将结果汇总到当前交付物后再进入下一步。

### 阶段0：明确本次任务

**步骤0：统一认知，定义任务范围**

- 开场强调：外种只是第一步，内收才决定最后结果
- 明确目标：围绕核心SKU，画出一条完整的「站外种草→站内收割」主路径
- 输入：品牌名、核心SKU、课程/项目主题、当前内容现状
- 输出：外种内收任务定义（一句话说明本次要完成什么）

### 阶段1-2：锁定核心SKU与站外触点

**步骤1：回看当前外种动作**
- 汇总品牌当前站外种草内容触点：小红书、抖音短视频、视频号、达人内容、社群内容等
- 输入：当前内容矩阵、达人合作、投放计划、社群动作
- 输出：站外触点清单

**步骤2：锁定核心SKU**
- 先选出一个最重要的核心SKU，围绕它设计主路径
- 原则：先画一条最重要路径，不要一开始就铺太多
- 输入：SKU清单、主推款、活动款、利润重点
- 输出：核心SKU确认表

**步骤3：梳理站外触点**
- 让AI帮助梳理当前最核心的站外触点和触点类型
- 原则：外种内收不是一个平台，而是一组触点系统
- 输入：内容平台、达人投放、内容形式、预算配置
- 输出：站外触点结构表

### 阶段3-5：心智→回淘→承接

**步骤4：判断站外内容种下了什么**
- 判断每个触点在种什么：搜索词、卖点、场景、品牌印象、活动预期
- 原则：不是泛泛做内容，而是明确「种什么」
- 输入：内容样本、关键词、卖点、场景、活动信息
- 输出：种下内容心智表

**步骤5：提炼回淘动作**
- 判断用户被种草后下一步更可能：去搜、进商品页、进店、看直播还是看活动页
- ⚠️ 关键：如果回淘动作不明确，整条链路就只是站外传播，不是外种内收
- 输入：内容导向、搜索词、平台路径、承接点
- 输出：回淘动作判断表

**步骤6：梳理站内承接点**
- 梳理商品页、店铺页、活动页、直播间、站内短内容模块等承接点
- 原则：不只是「有页」，而是要知道「由什么内容承接」
- 输入：商品页、店铺页、活动页、直播间、站内内容位
- 输出：站内承接点表

**步骤7：判断最终转化动作**
- 判断最终目标是收藏加购、下单、直播转化、会员沉淀还是私域转化
- 原则：外种内收的终点不是回流本身，而是最终的经营结果
- 输入：转化目标、运营目标、用户路径、承接逻辑
- 输出：最终转化动作表

### 阶段8：串成主路径

**步骤8：串成主路径**
- 将「站外触点→种下心智→回淘动作→站内承接点→最终转化动作」整合成一条路径
- 这是整个流程最核心的成果
- 输出：外种内收主路径图

使用以下模板格式：

```
| 站外触点 | 种下的内容心智 | 回淘动作 | 站内承接点 | 最终转化动作 | 备注 |
|---------|-------------|---------|-----------|------------|------|
| ...     | ...         | ...     | ...       | ...        | ...  |
```

### 阶段9-12：四项链路检查

按以下顺序逐一检查，每项输出一份检查表：

**步骤9：内容→搜索检查表（检查1）**
- 判断：内容是否种下明确搜索词（品牌词、品类词、卖点词、场景词）
- ⚠️ 用户看完不知道搜什么，是最常见的第一类断点
- 输入：内容文案、关键词、标题、评论区引导

**步骤10：搜索→承接一致性检查表（检查2）**
- 判断：商品标题、详情页、店铺页、活动页表达是否和站外内容一致
- ⚠️ 搜得出来但接不住，是最典型的流失点
- 输入：站外内容、标题、详情页、店铺页、活动页

**步骤11：店铺/活动页一体化检查表（检查3）**
- 判断：商品页、店铺页、活动页、直播间之间是否同一套逻辑
- ⚠️ 不是只有商品页能接，店铺页和活动页也必须接上
- 输入：店铺首页、活动页、直播间脚本、商品详情页

**步骤12：节奏同步检查表（检查4）**
- 判断：站外种草、站内搜索、承接页、直播、活动页节奏是否一致
- ⚠️ 节奏不同步，会让本来应该爆发的流量被浪费掉
- 输入：发布时间、活动时间、直播排期、搜索优化时间

四项检查使用统一模板格式：

```
| 检查项 | 当前情况 | 是否存在断点 | 最先该补什么 | 备注 |
|-------|---------|------------|------------|------|
| 检查1：内容有没有种下明确搜索 | ... | 是/否 | ... | ... |
| 检查2：搜索结果和承接页是否一致 | ... | 是/否 | ... | ... |
| 检查3：店铺页和活动页是否一体化 | ... | 是/否 | ... | ... |
| 检查4：站内外节奏是否同步 | ... | 是/否 | ... | ... |
```

### 阶段13-17：断点识别 → 修复 → 团队分工 → 复盘

**步骤13：找出链路最容易断的位置**
- 判断最明显的主断点和次断点
- 核心理念：链路不检查，内容越多，浪费越大
- 输出：链路断点识别表

**步骤14：形成链路图第一版**
- 将站外触点、心智种植、回淘动作、承接点、转化动作整合成一张图
- 这是最核心的成果表
- 输出：外种内收链路图初版

**步骤15：输出当前最先要补的环节**
- 判断最先值得修的环节：种词、承接页、节奏同步，还是店铺活动配合
- 原则：不要同时补全部，先抓最关键的一段
- 输入：断点识别结果、资源现状、目标
- 输出：首轮修复建议表

**步骤16：分配到团队动作**
- 拆成内容团队、电商团队、直播团队、运营/投放团队各自要补什么
- 原则：不拆到团队，链路图只是课堂成果
- 输出：团队动作分工表

使用以下模板格式：

```
| 团队 | 当前最该补哪一段 | 怎么补 | 负责人/时间 | 备注 |
|------|----------------|-------|-----------|------|
| 内容团队 | ... | ... | ... | ... |
| 电商团队 | ... | ... | ... | ... |
| 直播团队 | ... | ... | ... | ... |
| 运营/投放团队 | ... | ... | ... | ... |
```

**步骤17：建立后续复盘接口**
- 定义后续要看的信号：搜索增长、进店提升、承接页访问、直播导流、加购和转化变化等
- 原则：真正的链路图一定是可验证、可复盘的
- 输出：外种内收复盘指标表

## 三个关键认知（贯穿全程）

1. **不是站外没价值，而是站内没接住** -- 站外动作很多，但站内结果不同步，本质上不是外种问题，而是内收没接住。
2. **外种内收不是一句口号，而是一条完整路径设计** -- 内容先影响用户，再种下搜索/认知/场景，再把用户带回站内主战场。
3. **链路不检查，内容越多，浪费越大** -- 很多时候问题不在站外不够热，而在于整条链路从来没被系统检查过。

## 四个常见断点（优先排查）

1. 站外种草了，但没有种下清晰搜索词
2. 用户真的去搜了，但搜索结果页和商品页没接住
3. 内容导向和店铺承接不一致
4. 站外内容节奏和站内活动节奏不同步

## 五个最容易断的位置

| 断点 | 说明 |
|------|------|
| 断点1 | 内容没有种下明确的回淘动作 |
| 断点2 | 内容种下的是错误词或模糊词 |
| 断点3 | 承接页内容逻辑和站外内容逻辑不一致 |
| 断点4 | 店铺页和活动页没有形成一体化承接 |
| 断点5 | 站内外节奏断裂 |

## 老板版摘要模板

当完成全流程后，汇总输出老板版摘要，格式：

```
【一句话结论】外种只是第一步，内收才决定最后的结果。

【当前最重要的外种内收主路径】
（站外触点→心智→回淘→承接→转化的完整描述）

【最明显的断点位置】
（具体断点所在环节及现象描述）

【为什么这个断点最关键】
（对整体结果的影响分析）

【最先该补哪一段】
（优先级最高的一段及原因）

【本周最该先推进的修复动作】
（可立即执行的1-2项具体动作）
```

## 执行原则

- **先锁核心SKU** → 拉出站外触点 → 判断外面种下什么心智 → 判断用户回淘后会去哪里 → 明确站内由什么页面和内容接住 → 检查链路断点 → 形成一条主路径
- **先画一条最重要路径**，不要一开始就铺太多
- **先抓最关键的一段修复**，不要同时补全部
- **不拆到团队**，链路图只是课堂成果
- 每条结论必须可验证、每条修复建议必须可执行
