# AI内容—搜索—承接连接表 SOP

## 前置规则

执行前必须先读：`references/sop-main-table.md`（完整SOP步骤）、`references/module-understanding.md`（模块定义）、`references/core-questions.md`（核心问题）、`references/prompt-templates.md`（AI提示词模板）。

## 核心流程（16步）

### 阶段0：明确本次任务
- 统一认知：这一步不是写创意，而是补链路
- 明确目标：把内容→搜索→承接放进同一张表，形成完整链路
- 输入：品牌名、核心SKU、课程/任务主题
- **注意事项**：开场必须强调——内容不是孤立动作，而是链路第一环

### 阶段1：锁定核心SKU
- 从重点产品或货盘中，先选出最重要的1个SKU做演示
- 输入：SKU清单、活动主推款、店铺主推款
- **注意事项**：不要一上来填很多条，先填最重要的一条

### 阶段2：选定一条核心内容
- 选择最关键的一条内容路径（短视频/图文/达人内容/直播切片等）
- 输入：内容标题、脚本、链接、方向
- **注意事项**：先选一条最重要内容来打样

### 阶段3：明确这条内容在讲什么
- 让AI拆出：卖点、场景、人群、内容目的
- 使用提示词模板中"内容要素拆解"提示词
- 输出：内容要素拆解表
- **这是整张连接表的起点**

### 阶段4：明确内容目的
- 判断内容更偏：种草 / 解释 / 搜索引导 / 承接转化 / 直播引流
- 使用提示词模板中"内容目的判断"提示词
- **内容目的不清，后面搜索和承接都会乱**

### 阶段5：提炼关键词种子
- 从内容中提炼：品牌词、品类词、卖点词、场景词、需求词
- 使用提示词模板中"关键词种子提炼"提示词
- **内容不种词，后面很难形成搜索动作**

### 阶段6：推演用户看完后会搜什么
- 站在用户视角，判断看完后最可能搜什么词
- 使用提示词模板中"搜索导向推演"提示词
- **必须站在用户视角，不是品牌内部视角**

### 阶段7：分类搜索词
- 将搜索词拆成：品类词、品牌词、卖点词、场景词、需求词

### 阶段8：判断搜索是否能被接住
- 检查搜索后的：商品标题、店铺页、活动页、直播间是否能承接搜索词
- **搜出来不是对的表达，也等于链路断掉**

### 阶段9：明确承接落点
- 判断路径最终落：商品页 / 店铺页 / 活动页 / 直播间 / 私域
- 使用提示词模板中"承接落点判断"提示词
- **承接落点必须明确，不能模糊**

### 阶段10：检查承接页语言一致性
- 对比内容里的卖点/场景/关键词 与 商品页/店铺页/直播间表达是否一致
- 使用提示词模板中"语言一致性检查"提示词
- **如果前后不是一套语言，链路已经断了**

### 阶段11：找出链路断点
- 判断问题卡在：内容不清 / 搜索词不准 / 承接页没接住 / 表达不一致
- 使用提示词模板中"链路断点诊断"提示词
- **不要模糊说效果不好，要说清楚断在哪**

### 阶段12：补齐链路建议
- 给出补齐建议：内容怎么改、词怎么换、承接页怎么接、直播间怎么承接
- 使用提示词模板中"补链路建议"提示词
- **现场一定要落到"怎么补"**

### 阶段13：形成连接表第一版
- 将"内容—搜索—承接"三段整合成一条标准链路
- 输出：连接表初版（使用 `assets/connect-table-template.md` 模板）

### 阶段14：输出首轮优先动作
- 判断先改：内容 / 先补关键词 / 先补商品页 / 先补直播间
- **一次只抓最关键的一两个问题**

### 阶段15：分配到团队动作
- 拆成：内容团队、电商团队、直播团队、客服团队、设计/页面团队各自要补的动作
- 使用提示词模板中"团队动作拆解"提示词
- **不拆到团队，连接表就只是课堂表格**

### 阶段16：建立复盘接口
- 明确后续复盘看什么：搜索提升、点击率、承接页停留、咨询率、成交率等
- **连接表不是一次性填完，而是后续要验证**

## 输出清单（12项）

按流程执行后，应输出以下完整成果：

| 序号 | 交付物 | 用途 |
|------|--------|------|
| 1 | 核心内容样本表 | 明确本次分析哪条内容路径 |
| 2 | 内容要素拆解表 | 明确卖点、场景、人群、内容目的 |
| 3 | 搜索导向词表 | 明确用户看完后最可能搜什么 |
| 4 | 搜索词分类表 | 品类词/品牌词/卖点词/场景词/需求词 |
| 5 | 承接落点表 | 明确用户搜到后进入哪里 |
| 6 | 语言一致性检查表 | 检查内容、搜索、承接是否一套语言 |
| 7 | 链路断点结论表 | 明确链路断在哪一环 |
| 8 | 链路优化建议表 | 明确该怎么补链路 |
| 9 | **内容—搜索—承接连接表** | 现场最核心的标准输出物 |
| 10 | 团队动作分工表 | 让各团队真正接起来 |
| 11 | 老板版摘要表 | 一句话结论版 |
| 12 | 复盘指标表 | 后续验证链路是否变顺 |

## 执行规则

1. **严格按16步顺序执行**，每步必须确认输出后再进入下一步
2. **每个AI提示词必须按模板完整复制**给用户，不可精简
3. **所有输出必须结构化**：优先使用表格形式
4. **每步输出后等待用户确认**，确认后再进行下一步
5. **先锁定1个SKU + 1条内容路径**，不要一次性填很多条
6. **最终必须将全部12项交付物汇总输出**

## 核心教学金句

> 1. 内容不是起点，内容是链路的第一环。
> 2. 一条真正能为增长服务的内容，不只负责被看见，还要负责把"看见→搜索→承接→成交"串起来。
> 3. 连接表补不出来，说明现在还只是"内容创意图"，还不是"增长链路图"。
