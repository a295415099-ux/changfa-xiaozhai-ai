# 优质素材筛选 Skill SOP

## 核心认知

**优质素材不是「播放高的素材」，而是能种草、能记词、能回流、能承接、能转化、能复用的素材。**

播放高 ≠ 值得放大。很多品牌判断素材好坏只看播放量、点赞量、达人数据或阅读量，但真正值得投放放大的素材需要从六个维度系统评估。

---

## 执行流程（18步）

严格按照以下 18 个步骤执行，每完成一步输出对应的结果，再进入下一步。

### 阶段 0：明确本次任务

**目标**：统一认知——这一步不是找播放最高素材，而是筛真正值得放大的素材。

**动作**：向用户确认任务目标、品牌信息、待筛选素材范围。

**输出**：素材筛选任务定义（品牌名、筛选范围、筛选目标、可用数据源）。

**注意事项**：开场要强调「播放高，不等于值得放大」。

---

### 阶段 1：回收素材基础资料

**目标**：先把待筛选素材和相关数据拉出来。

**动作**：请用户提供以下数据（至少包含前3项）：
- 待筛选素材列表（短视频/笔记/达人内容/投放素材）
- 互动数据（播放/点赞/收藏/评论/分享/完播率）
- 进店与转化数据（点击/进店/加购/成交/咨询）
- 评论区数据
- 搜索词数据
- 商品页信息

**输出**：素材基础资料表（汇总所有待筛选素材及已有数据）。

**注意事项**：没有基础数据，筛选会流于主观判断。

---

### 阶段 2：检查素材基础关联

**目标**：先判断内容和产品关联是否足够强。

**动作**：检查每条素材：
- 用户看完后是否记住产品？
- 用户看完后是否记住要解决的问题？
- 用户看完后是否记住可搜索的关键词？
- 还是只「看了个热闹」？

**输出**：素材关联性检查表。

**注意事项**：内容火了但用户记不住产品，不建议直接放大。

---

### 阶段 3：评估种草力（1-5分）

**目标**：判断用户是否被真正种草。

**评分依据**（详见 `references/scoring-criteria.md` 第1节）：
- 评论区代入感表达
- 收藏/点赞比
- 主动私信询问
- 完播率/停留时长
- 需求信号（"哪里买"、"适合我吗"等）

**输出**：种草力评分表（每条素材 1-5 分 + 评分理由）。

**注意事项**：种草力核心是用户是否觉得「这说的是我」。

---

### 阶段 4：评估关键词承接力（1-5分）

**目标**：判断素材是否自然强化了品牌想沉淀的词。

**评分依据**（详见 `references/scoring-criteria.md` 第2节）：
- 标题关键词
- 口播/文案关键词植入
- 评论区关键词讨论
- 用户搜索词匹配度
- 商品标题与素材关键词一致性

**输出**：关键词力评分表（每条素材 1-5 分 + 评分理由）。

**注意事项**：关键词是内容回流和长期资产的关键层。

---

### 阶段 5：评估转化潜力（1-5分）

**目标**：判断素材有没有把用户推向进店、咨询、加购、成交。

**评分依据**（详见 `references/scoring-criteria.md` 第3节）：
- 点击率（CTR）
- 商品卡点击/进店UV
- 加购率
- 咨询发起量
- 成交辅助数据

**输出**：转化力评分表（每条素材 1-5 分 + 评分理由）。

**注意事项**：播放高但不转化的素材不应直接放大。

---

### 阶段 6：评估互动质量（1-5分）

**目标**：判断评论区是否出现真实需求而不是泛互动。

**评分依据**（详见 `references/scoring-criteria.md` 第4节）：
- 「适合谁」类型提问
- 「怎么选」类型提问
- 「搜什么」类型提问
- 「和什么区别」类型提问（竞品对比）
- 「有没有直播间」类型提问

泛互动（纯表情、简单点赞、"好看"）不算高质量互动。

**输出**：互动质量评分表（每条素材 1-5 分 + 评分理由）。

**注意事项**：泛互动不等于高质量互动，关键看是否有购买问题。

---

### 阶段 7：评估承接匹配（1-5分）

**目标**：判断店铺页面能不能接住这条内容。

**评分依据**（详见 `references/scoring-criteria.md` 第5节）：
- 商品标题是否包含素材关键词
- 主图是否呼应素材种草点
- 详情页首屏与素材一致性
- 客服话术准备情况
- 直播间承接能力

**输出**：承接匹配评分表（每条素材 1-5 分 + 评分理由）。

**注意事项**：店铺承接不住，再好的素材也难形成稳定转化。

---

### 阶段 8：评估复用价值（1-5分）

**目标**：判断这条素材能不能拆成更多资产。

**评分依据**（详见 `references/scoring-criteria.md` 第6节）：
- 可改为短视频（不同时长/不同开头）
- 可改为小红书笔记
- 可作为直播口播素材
- 可提取为详情页模块
- 可沉淀为客服话术/FAQ
- 可作为达人Brief模板
- 可作为投放素材

**输出**：复用价值评分表（每条素材 1-5 分 + 评分理由）。

**注意事项**：可复制、可二创、可沉淀，才是长期优质素材。

---

### 阶段 9：汇总素材总分

**目标**：把6个维度收口成一张评分表。

**动作**：将阶段3-8的六维评分汇总。如有素材数据JSON格式，可使用 `python3 scripts/scoring_tool.py <materials.json>` 自动生成评分表和分级。

**输出**：素材筛选评分表（含六维分 + 总分 + 等级）。

**注意事项**：这是本 Skill 的核心打分环节。

---

### 阶段 10：做素材分级

**目标**：按A/B/C/D分级标准判断处理方式。

根据总分划分等级（详见 `references/grading-standards.md`）：

| 等级 | 总分 | 处理方式 |
|------|------|----------|
| A级 | 25-30分 | 优先放大 |
| B级 | 20-24分 | 优化后放大 |
| C级 | 15-19分 | 观察/待定 |
| D级 | 15分以下 | 淘汰/不再放大 |

**输出**：素材分级表。

**注意事项**：分级能直接指导后续投放与内容生产。

---

### 阶段 11：识别A级素材特征

**目标**：总结真正值得放大的素材共性。

**动作**：提炼A级素材的5大共性特征：
1. 用户精准（评论精准匹配目标人群）
2. 痛点明确（用户看完就知道解决什么问题）
3. 关键词清晰（三端关键词统一且有搜索导向）
4. 承接顺畅（素材→搜索→商品页路径无缝）
5. 可复制（素材结构清晰，可拆解可二创）

**输出**：优质素材特征表。

**注意事项**：这一步帮助团队建立长期筛选模型。

---

### 阶段 12：识别不建议放大素材

**目标**：总结哪些素材不值得直接继续投。

**动作**：整理D级素材的5种典型特征：
1. 只有播放高没有关键词
2. 只有泛互动没有真实购买问题
3. 内容与产品关联弱
4. 店铺承接不住
5. 无法复用

**输出**：不建议放大素材样本表。

**注意事项**：反面样本同样重要，可以避免团队重复踩坑。

---

### 阶段 13：做放大前检查

**目标**：在投放前逐项检查关键词、承接、客服、评论区和复用可能。

**动作**：逐项检查8项内容（详见 `references/grading-standards.md` 检查清单部分）：
1. 关键词统一
2. 商品标题承接
3. 详情页首屏承接
4. 客服准备
5. 评论区问题整理
6. 投放指标确认
7. 二次剪辑可能
8. 直播间承接

**输出**：素材放大前检查清单。

**注意事项**：这一步是素材放大前必须过的一关。

---

### 阶段 14：形成《素材筛选评分表》正式版

**目标**：把全部筛选结果沉淀成正式工具表。

**动作**：将六大维度评分、总分、素材等级、处理方式整合为完整版评分表。

**输出**：素材筛选评分表正式版。

---

### 阶段 15：输出当前最值得放大的素材

**目标**：帮团队知道先放大哪几条内容。

**动作**：从全部素材中筛出最值得优先放大的1-3条A级/B级素材，给出具体理由和建议动作。

**输出**：优先放大素材清单（1-3条，含理由和分工建议）。

**注意事项**：这样团队不会因为素材太多而失焦。

---

### 阶段 16：分配到团队执行

**目标**：让筛选结果进入内容、投放、电商、客服团队执行。

**动作**：按照四团队分工表分配任务（详见 `references/grading-standards.md` 处理动作速查表）：

| 等级 | 投放团队 | 内容团队 | 电商团队 | 客服团队 |
|------|----------|----------|----------|----------|
| A级 | 加大预算投放 | 二创多版本分发 | 优化承接页 | 准备FAQ话术 |
| B级 | 小预算测试 | 针对性优化短板 | 配合优化承接 | 补充对应话术 |
| C级 | 不投放 | 观察数据变化 | 视情况调整 | 关注相关咨询 |
| D级 | 不投放 | 记录为反面样本 | 不调整 | 不准备 |

**输出**：团队执行分工表。

**注意事项**：不拆到团队，筛选表很快会失效。

---

### 阶段 17：建立滚动复盘机制

**目标**：让素材筛选能力持续迭代。

**动作**：设计后续如何根据以下数据持续更新筛选标准：
- 投放ROI表现
- 搜索回流数据
- 成交承接数据
- 复用效果反馈

**输出**：素材筛选复盘机制表（含复盘周期、复盘指标、更新方式）。

**注意事项**：优质素材筛选是动态能力，不是一次性动作。

## 使用工具

### 评分表生成工具

当用户提供了多条素材的评分数据后，可以使用内置脚本快速生成评分表：

```bash
python3 scripts/scoring_tool.py <materials.json>
```

`materials.json` 格式见下方示例。脚本会输出完整的评分表、优先放大清单和检查清单。

**查看评分维度**：
```bash
python3 scripts/scoring_tool.py --dimensions
```

**查看单项分级**：
```bash
python3 scripts/scoring_tool.py --classify <score>
```

### materials.json 格式示例

```json
{
  "materials": [
    {
      "name": "达人A种草视频-敏感肌护肤",
      "scores": {
        "seeding_power": 5,
        "keyword_power": 4,
        "conversion_power": 4,
        "interaction_quality": 5,
        "landing_match": 4,
        "reuse_value": 5
      }
    }
  ],
  "priority_list": true,
  "checklists": false
}
```

评分字段说明：
- `seeding_power` - 种草力
- `keyword_power` - 关键词力
- `conversion_power` - 转化力
- `interaction_quality` - 互动质量
- `landing_match` - 承接匹配
- `reuse_value` - 复用价值
