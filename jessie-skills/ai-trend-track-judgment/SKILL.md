# AI趋势赛道判断表 SOP

将品牌的"短期内容动作"升级为"赛道方向判断"，系统判断哪些方向正在上升、哪些已经拥挤、哪些值得提前卡位。

## 核心框架：三层趋势判断

| 层面 | 回答的问题 | 判断方式 |
|------|-----------|---------|
| 用户趋势 | 用户最近更关注什么 | 看用户需求与表达方式是否在变化 |
| 内容趋势 | 平台最近更愿意放大什么 | 看哪些内容形式和表达方式正在变高效 |
| 赛道趋势 | 哪些细分方向升温、哪些已红海 | 看行业格局变化和竞争状态 |

## 执行流程

按以下18步SOP逐步执行。详细流程见 [references/sop-workflow.md](references/sop-workflow.md)。

### 快速执行指引

1. **明确任务** → 告知用户这是"赛道方向判断"，不是单条内容判断
2. **收信息** → 向用户收集：品牌名、赛道、SKU、竞品、已有洞察数据
3. **用户趋势** → 基于评论区/问大家/客服/反馈，判断用户关注点变化
4. **内容趋势** → 基于平台热点/高表现内容/同行爆款，判断内容形式和效率变化
5. **赛道趋势** → 基于行业信息/竞品布局，判断方向升温和红海状态
6. **三类分类** → 强行归类为：值得追 / 需谨慎 / 提前卡位（每类至少1个）
7. **匹配度** → 判断品牌匹配度和节点匹配度
8. **动作建议** → 先观察 / 先测试 / 先卡位 / 先投入
9. **建立验证** → 定义后续观察信号和验证指标

### 核心金句

> "趋势判断，不是追热点，而是看什么值得追、什么要谨慎、什么适合卡位。"

## 输出要求

每个品牌至少输出：
- **1个**值得追的趋势方向
- **1个**需要谨慎跟的热闹方向
- **1个**适合自己品牌提前卡位的方向

## 关键原则

1. **先界定赛道再判断** — 不先定义赛道和细分方向，趋势判断容易泛化
2. **基于真实数据** — 用户趋势尽量基于真实表达而非主观感受
3. **内容趋势看效率** — 不是"风格新"，而是"效率更高"
4. **强行三类分类** — 不要只写观察描述，要明确分类为追/谨慎/卡位
5. **趋势判断要验证** — 定义后续观察信号，不是一次性拍脑袋结论
6. **最终回到动作** — 趋势判断的终点是能做什么，不是看到什么

## 执行注意事项

- 趋势判断是中观判断，不是短期选题表
- 用户趋势基于真实表达（评论区、问大家、客服问题），不是主观猜测
- 内容趋势的核心判断标准是"效率更高"而非"形式更新"
- 不同赛道适合的内容形式不同，不可套用
- 第7步「赛道趋势判断」是整张表最核心的判断步骤
- 值得追≠盲追，要看品牌能不能承接
- 热闹≠机会，很多方向只是看起来热
- 卡位方向关键是"现在做不挤，未来有可能放大"
- 最终必须强行三类分类，不能只写观察
- 趋势判断必须伴随验证机制（用户反应/内容表现/搜索增长/竞品跟进/平台放大）
