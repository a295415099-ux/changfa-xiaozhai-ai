# AI店铺承接优化动作清单 SOP

## 概述

本技能将"店铺承接优化"从一句空话拆成一组可执行动作。核心逻辑：承接不是只改商品页，而是优化标题首屏、商品页卖点、店铺首页、活动页与直播间、利益点与转化动作五套系统。最终输出可落地推进的动作清单。

**一句话：承接不是一句空话，而是一组必须被拆出来、立刻推进的动作。**

## 核心认知（每次执行前确认）

1. **承接不是只改商品页** — 真正接住用户的是整套店铺承接系统，不是单页优化。
2. **店铺承接优化不是一句空话，而是一组动作** — 拆成标题、首屏、商品页、首页、活动页、直播间、利益点等具体动作。
3. **真正的承接优化，不是"多"，而是"顺"** — 用户被内容打动后，能不能顺着路径继续往前走，才是关键。

## 五大优化模块与判断标准

| 模块 | 判断问题 | 标准 |
|------|---------|------|
| 标题与首屏优化 | 用户进来第一眼有没有被接住 | 标题接住搜索词和卖点词，首屏继续放大购买理由 |
| 商品页卖点承接 | 商品页是否继续内容里的那套逻辑 | 卖点表达一致，层级清楚 |
| 店铺首页承接 | 首页是否延续内容逻辑 | 不是无序商品堆砌，而是接住主场景和主逻辑 |
| 活动页与直播间承接 | 用户被打动后能否快速找到下一步 | 活动页、直播间继续承接内容引导 |
| 利益点与转化动作 | 用户是否有明确理由和路径往前走 | 优惠、赠品、机制、按钮、导流路径足够清晰 |

## 执行流程（20步SOP）

详细流程见 `references/sop-workflow.md`。执行时按以下阶段推进：

### 前置阶段（0-3）: 准备

- **阶段0**：明确任务 → 输出「店铺承接优化任务定义」
- **阶段1**：回看当前承接现状 → 输出「当前承接现状表」
- **阶段2**：锁定核心SKU → 输出「核心SKU确认表」
- **阶段3**：回收前面内容路径 → 输出「内容输入信息表」

### 检查与诊断（4-13）: 五大模块逐一检查

**标题与首屏（4-5）**：
- 检查标题是否接住搜索词和卖点词，首屏是否继续放大购买理由
- 输出「标题与首屏检查表」「标题与首屏优化动作表」

**商品页卖点（6-7）**：
- 检查卖点层级是否清楚、与外部内容是否一致
- 输出「商品页卖点检查表」「商品页卖点优化动作表」

**店铺首页（8-9）**：
- 检查首页是否围绕内容逻辑和核心SKU延续表达
- 输出「店铺首页承接检查表」「店铺首页优化动作表」

**活动页与直播间（10-11）**：
- 检查活动页/直播间的入口、话术、利益点是否与内容目标一致
- 输出「活动页与直播间检查表」「活动页与直播间优化动作表」

**利益点与转化（12-13）**：
- 检查优惠、赠品、机制、按钮、引导路径是否清晰
- 输出「利益点与动作检查表」「利益点优化动作表」

### 汇总与决策（14-17）: 形成行动清单并做优先级判断

- **阶段14**：汇总全部动作 → 「店铺承接优化动作清单」
- **阶段15**：筛出Top3立即优化动作 → 「Top3立即优化动作表」
- **阶段16**：确定618前必做动作 → 「618前必做动作表」
- **阶段17**：确定60天持续优化动作 → 「60天持续优化动作表」

### 落地与复盘（18-19）: 进入执行

- **阶段18**：分配到团队 → 「团队动作分工表」
- **阶段19**：建立复盘接口 → 「承接优化复盘指标表」

## 最终输出物

完成后必须输出以下核心交付物：

1. **店铺承接优化动作清单**（★★★ 核心表）
2. **Top3立即优化动作表**
3. **618前必做动作表**
4. **60天持续优化动作表**
5. **团队动作分工表**
6. **老板版摘要**（给老板看的一页纸）

## 执行注意

- **始终先看清现状** → 没有现状就直接给动作会很空
- **先优化一条承接路径** → 锁一个核心SKU，不要全部铺开
- **承接必须接前面的内容逻辑** → 先回收内容里种下的搜索词、卖点词、场景
- **最终要逼团队聚焦** → 不是列一堆"都重要"，而是明确Top3、必做项、持续项
- **拆到团队** → 不拆到具体团队，动作清单很难真正推进
- **建立复盘指标** → 搜索回流、商品页停留、店铺页访问、直播间点击、加购率、下单率
