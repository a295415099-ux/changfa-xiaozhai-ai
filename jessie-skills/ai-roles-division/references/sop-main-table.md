# SOP主表完整内容（18步详解）

## 阶段0：明确本次任务
| 项目 | 内容 |
|------|------|
| 任务目标 | 统一认知：课程结果不是靠记住，而是靠回店后的协同推进 |
| 具体动作 | 明确目标：围绕"老板/品牌/电商"三角色，形成一张真实可执行的分工表 |
| 输入内容示例 | 品牌名、参课人员、课程主题、大促目标 |
| 输出结果 | 三角色分工任务定义 |
| 注意事项 | 开场一定要强调：角色不清，前面所有内容最后都会停在会议纪要里 |

## 阶段1：回看课程输出物
| 项目 | 内容 |
|------|------|
| 任务目标 | 先把前面已经做出来的内容接回来 |
| 具体动作 | 汇总内容流量地图、内容优先级、断链识别、外种内收、承接优化、表达模板等成果 |
| 输入内容示例 | 地图表、动作表、断链表、模板表 |
| 输出结果 | 课程成果汇总表 |
| 注意事项 | 分工表不是孤立存在，而是承接前面所有成果 |

## 阶段2：明确真实参课角色
| 项目 | 内容 |
|------|------|
| 任务目标 | 确定这次回店后到底是谁推进 |
| 具体动作 | 不写虚角色，而是把这次真实参课的老板、品牌负责人、电商负责人放进表里 |
| 输入内容示例 | 真实姓名、岗位、当前职责 |
| 输出结果 | 真实角色确认表 |
| 注意事项 | 一定要按真实人员填，不能写"理论上的岗位" |

## 阶段3：定义老板角色的核心任务
| 项目 | 内容 |
|------|------|
| 任务目标 | 看老板回店后具体负责什么 |
| 具体动作 | 明确老板/创始人/一号位负责定经营优先级、定预算资源、定主战SKU和主战平台、拍板关键取舍 |
| 输入内容示例 | 经营目标、预算、资源、平台布局 |
| 输出结果 | 老板职责表 |
| 注意事项 | 老板不负责天天盯执行，但必须负责关键拍板 |

## 阶段4：定义品牌角色的核心任务
| 项目 | 内容 |
|------|------|
| 任务目标 | 看品牌负责人具体负责什么 |
| 具体动作 | 明确品牌/市场负责人负责统一内容表达、管理三类词系统、管理内容方向、外部种草逻辑和品牌内容标准 |
| 输入内容示例 | 三类词、表达模板、内容方向、平台内容逻辑 |
| 输出结果 | 品牌职责表 |
| 注意事项 | 品牌角色的核心是"统一表达"，不是只写内容 |

## 阶段5：定义电商角色的核心任务
| 项目 | 内容 |
|------|------|
| 任务目标 | 看电商负责人具体负责什么 |
| 具体动作 | 明确电商/运营负责人负责搜索承接、商品页与店铺页承接、活动页与直播协同、转化路径优化、数据反馈 |
| 输入内容示例 | 搜索词、商品页、活动页、直播间、站内数据 |
| 输出结果 | 电商职责表 |
| 注意事项 | 电商角色的核心是"接结果"，不是只盯GMV数字 |

## 阶段6：判断三角色缺谁最危险
| 项目 | 内容 |
|------|------|
| 任务目标 | 帮团队看清如果少一个角色会出什么问题 |
| 具体动作 | 让AI帮助判断少了老板、品牌或电商任一角色，回店后最容易断在哪 |
| 输入内容示例 | 三角色职责、当前组织现状、推进难点 |
| 输出结果 | 角色缺失风险表 |
| 注意事项 | 这一步帮助团队理解为什么"1+2模式"不能少人 |

## 阶段7：明确老板负责的拍板事项
| 项目 | 内容 |
|------|------|
| 任务目标 | 避免老板角色停留在"知道方向" |
| 具体动作 | 让AI梳理：哪些事项必须由老板拍板，比如预算、资源、平台轻重缓急、主战SKU、关键取舍 |
| 输入内容示例 | 当前项目关键决策、预算、平台策略 |
| 输出结果 | 老板拍板事项表 |
| 注意事项 | 老板角色不能空泛，一定要落到具体拍板事项 |

## 阶段8：明确品牌负责人负责的统一项
| 项目 | 内容 |
|------|------|
| 任务目标 | 防止表达系统再次分裂 |
| 具体动作 | 明确品牌负责人具体负责三类词统一、内容表达标准、内容优先级、外部内容输出逻辑 |
| 输入内容示例 | 词表、内容标准、优先级表、外种逻辑 |
| 输出结果 | 品牌统一事项表 |
| 注意事项 | 品牌负责人是"统一表达系统"的核心角色 |

## 阶段9：明确电商负责人负责的承接项
| 项目 | 内容 |
|------|------|
| 任务目标 | 防止站内承接重新断掉 |
| 具体动作 | 明确电商负责人具体负责搜索承接、店铺承接、活动页/直播间承接、站内数据反馈 |
| 输入内容示例 | 搜索承接表、店铺承接优化表、直播承接表 |
| 输出结果 | 电商承接事项表 |
| 注意事项 | 电商负责人不只是接转化，还要把品牌表达接进站内 |

## 阶段10：梳理三方共同协同事项
| 项目 | 内容 |
|------|------|
| 任务目标 | 明确哪些事必须三方一起看 |
| 具体动作 | 让AI梳理老板/品牌/电商三方共同协同事项：每周看什么、什么节点一起开会、谁提需求、谁催进度、谁复盘 |
| 输入内容示例 | 周报维度、会议节奏、项目节点、复盘要求 |
| 输出结果 | 三方协同事项表 |
| 注意事项 | 这一步最关键，它决定回店后会不会继续散 |

## 阶段11：判断谁先提需求
| 项目 | 内容 |
|------|------|
| 任务目标 | 避免流程入口混乱 |
| 具体动作 | 明确在不同阶段，到底是老板先提经营需求、品牌先提内容需求，还是电商先提承接需求 |
| 输入内容示例 | 经营节奏、品牌节奏、电商承接问题 |
| 输出结果 | 需求发起规则表 |
| 注意事项 | 如果谁都能随时发起需求，流程很快会重新乱掉 |

## 阶段12：判断谁来催进度
| 项目 | 内容 |
|------|------|
| 任务目标 | 防止所有事情停在"都知道" |
| 具体动作 | 明确项目推进里谁来做节奏推进与催办 |
| 输入内容示例 | 团队组织、岗位分工、时间节点 |
| 输出结果 | 进度推进责任表 |
| 注意事项 | 没有催进度的人，项目一定会越来越松散 |

## 阶段13：判断谁来做结果复盘
| 项目 | 内容 |
|------|------|
| 任务目标 | 防止动作做完后没人沉淀经验 |
| 具体动作 | 明确谁来负责看结果、做复盘、提下一轮优化建议 |
| 输入内容示例 | 数据反馈、内容结果、店铺结果、会议机制 |
| 输出结果 | 结果复盘责任表 |
| 注意事项 | 真正让课程跑出结果的，不只是做动作，而是持续复盘 |

## 阶段14：形成三角色分工表初版
| 项目 | 内容 |
|------|------|
| 任务目标 | 把分工真正沉淀成标准表单 |
| 具体动作 | 将老板、品牌、电商三角色的职责，以及三方协同事项整合成一张表 |
| 输入内容示例 | 前面所有结果 |
| 输出结果 | 老板/品牌/电商三角色分工表初版（第四张关键成果表） |
| 注意事项 | 这是核心交付物 |

## 阶段15：输出回店后第一周协同动作
| 项目 | 内容 |
|------|------|
| 任务目标 | 让角色分工马上进入执行 |
| 具体动作 | 让AI给出这三类角色回店后第一周最应该分别做什么 |
| 输入内容示例 | 分工表、大促目标、当前工作节奏 |
| 输出结果 | 第一周协同动作表 |
| 注意事项 | 角色分工不是看完就结束，必须立刻进入第一周动作 |

## 阶段16：输出最关键协同会议机制
| 项目 | 内容 |
|------|------|
| 任务目标 | 确保角色之间后续能持续对齐 |
| 具体动作 | 让AI设计每周看什么、在哪个节点开会、三方怎么对齐的会议机制 |
| 输入内容示例 | 工作节奏、节点安排、项目周期 |
| 输出结果 | 协同会议机制表 |
| 注意事项 | 不建立会议机制，三角色很快又会重新分开跑 |

## 阶段17：建立后续复盘接口
| 项目 | 内容 |
|------|------|
| 任务目标 | 让分工表后面能真正被验证 |
| 具体动作 | 设定后续要观察的协同信号：任务推进率、表达统一度、承接匹配度、会议执行率、复盘完成率等 |
| 输入内容示例 | 指标口径、节奏、组织要求 |
| 输出结果 | 三角色协同复盘指标表 |
| 注意事项 | 分工表不是"写清楚就结束"，而要能持续被检查和迭代 |
