# AI内容断链检查表SOP

核心命题：**不是内容没价值，而是内容后的动作没接住。**

帮助品牌系统检查从"内容被看见"到"回到站内主战场"的链路是否断裂。

## 适用对象

品牌老板、商家操盘手、内容团队、电商团队、直播团队、客服团队、培训讲师。

## 四种断链类型

| 断链类型 | 典型表现 | 本质问题 |
|----------|----------|----------|
| 关键词断链 | 内容有热度，但用户不知道下一步搜什么 | 内容没有种下明确关键词 |
| 搜索承接断链 | 用户去搜了，但搜出来的页接不住 | 站内页面没有准备好 |
| 利益点断链 | 用户进来了，但没有明确购买理由 | 承接页没讲透为什么买、现在为什么买 |
| 节奏错位断链 | 内容先做了，但搜索、直播、活动页没跟上 | 内容、搜索、承接节奏不同步 |

## 执行流程（17步SOP）

执行前必须确认品牌名、核心SKU、一条关键内容路径。建议使用方式：锁定1个核心SKU与1条关键内容路径 → 逐项检查 → 输出修复动作。

### 阶段0：明确任务
- 统一认知：这一步不是看内容热不热，而是看链路有没有断。
- 输入：品牌名、核心SKU、课程主题。
- 输出：断链检查任务定义。

### 阶段1-2：锁定检查对象
- **步骤1**：锁定核心SKU（先检查一个最关键的，不要铺太多）
- **步骤2**：锁定对应内容样本（选最典型的一条短视频/图文/达人内容/直播切片）
- 输出：核心SKU确认 + 内容样本确认表

### 阶段3-5：关键词与搜索导向检查
- **步骤3**：判断内容是否种下关键词 → 输出《关键词种植检查表》
  - 检查有没有清晰种下品牌词、品类词、卖点词、场景词
  - 如果用户看完不知道搜什么 → 第一类断点
- **步骤4**：推演用户看完后的动作 → 输出《用户下一步动作表》
  - 站在用户视角判断：搜词 / 进主页 / 点商品 / 进直播 / 直接离开
- **步骤5**：检查搜索词是否清晰 → 输出《搜索导向清晰度表》
  - 词不清晰，用户喜欢也没法进一步行动

### 阶段6-7：站内承接检查
- **步骤6**：检查站内搜索承接 → 输出《搜索承接检查表》
  - 检查商品标题、详情页、店铺页、活动页、直播标题是否接住内容种下的词
  - 搜到了但没接住 → 第二类断点
- **步骤7**：检查页面表达一致性 → 输出《表达一致性检查表》
  - 对比内容里的卖点/场景/关键词与页面表达是否统一
  - 表达脱节，用户快速流失

### 阶段8-9：利益点与转化动作
- **步骤8**：检查承接页利益点 → 输出《利益点检查表》
  - 判断是否讲清楚为什么买、现在为什么买
  - 进来了但没理由买 → 第三类断点
- **步骤9**：检查承接页转化动作 → 输出《转化动作检查表》
  - 检查是否有清晰CTA：加购、领券、进直播间、咨询客服等
  - 没有转化动作，用户停留但不成交

### 阶段10：节奏同步检查
- **步骤10**：检查节奏是否同步 → 输出《节奏同步检查表》
  - 判断内容发布时间与搜索优化、直播排期、活动页上线是否同步
  - 时间错位会浪费流量 → 第四类断点

### 阶段11-12：归因与优先级
- **步骤11**：判断断链类型 → 输出《断链类型归类表》
  - 归类到四大断链类型，先分类后解决
- **步骤12**：找出最关键断点 → 输出《核心断点结论表》
  - 判断最影响结果的1-3个核心断点，不要平均用力

### 阶段13-17：修复与落地
- **步骤13**：输出断链修复建议 → 输出《断链修复建议表》
  - 内容怎么补词、页面怎么接、利益点怎么补、节奏怎么调
- **步骤14**：形成断链检查表第一版 → 输出《内容断链检查表》
  - 下午第一阶段最重要的收口表
- **步骤15**：输出首轮优先修复动作 → 输出《首轮优先修复表》
  - 至少3个最关键内容断链问题，排先后顺序
- **步骤16**：分配到团队动作 → 输出《团队补位动作表》
  - 拆到内容团队/电商团队/直播团队/客服团队各自的补位动作
- **步骤17**：建立后续复盘接口 → 输出《修复后复盘指标表》
  - 明确后续观察指标：搜索增长、页面停留、咨询率、加购率、成交率

## 核心检查点

执行时重点回答四个问题：
1. 内容有没有种下清晰关键词？（看完不知道搜什么 → 断点）
2. 搜索出来的页是不是接得住？（页面表达跟内容脱节 → 断点）
3. 承接页有没有明确利益点和转化动作？（没有清晰购买理由 → 断点）
4. 内容发布节奏和站内承接节奏是否同步？（时间错位 → 断点）

## 课堂/现场注意事项

- 一开始就讲清楚核心逻辑：不是内容没价值，而是内容后的动作没接住
- 先检查一个最关键的SKU，不要一开始铺太多
- 先找最典型的一条内容来查断点
- 检查表不是讨论理想打法，而是马上识别当前真实存在的断点
- 一定要逼商家做优先级，至少找3个最关键问题
- 断链问题必须拆到团队，不然不会动
- 检查不是终点，后续要验证问题是否修复
