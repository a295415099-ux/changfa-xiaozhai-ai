# 老板版摘要模板

## 模板格式

| 项目 | 内容 |
|------|------|
| 一句话结论 | 知道要做什么还不够，真正决定结果的是：什么时候做、谁来做、先后顺序是什么。 |
| 回店后30天最关键动作 | |
| 最容易延误的动作 | |
| 哪些节点必须老板盯 | |
| 本周最该先推进的动作 | |
| 甘特表后续怎么滚动更新 | |

## 列说明

- **一句话结论**：固定话术，强调甘特表的核心价值
- **回店后30天最关键动作**：从甘特表中提炼出30天内最核心的3-5个关键动作
- **最容易延误的动作**：列出最可能因协同、资源、时间冲突而延期的Top3动作
- **哪些节点必须老板盯**：列出老板必须参与或拍板的关键节点
- **本周最该先推进的动作**：从甘特表筛出本周最优先的3-5个动作
- **甘特表后续怎么滚动更新**：说明更新机制（谁负责、什么频率、什么节点触发）

## 使用场景

面向老板/创始人/一号位的简化摘要输出，让老板快速了解：
1. 接下来30天最关键要推进什么
2. 哪些地方最容易出问题需要提前关注
3. 哪些节点必须自己参与
4. 本周团队在做什么
5. 这个甘特表后续怎么保持活性
