# 甘特表滚动更新机制模板

## 模板格式

| 更新项 | 每周怎么更新 | 谁来更新 | 更新频率/节点 | 备注 |
|-------|------------|---------|-------------|------|
| 新增动作 |            |         |             |      |
| 延误动作调整 |            |         |             |      |
| 负责人变更 |            |         |             |      |
| 关键节点复盘 |            |         |             |      |
| 低优先级动作删除/后移 |            |         |             |      |

## 列说明

- **更新项**：五类需要滚动更新的内容
  - 新增动作：执行过程中新识别的任务
  - 延误动作调整：因延误需要调整时间或资源的动作
  - 负责人变更：人员变动导致的负责人调整
  - 关键节点复盘：对已完成关键节点的复盘
  - 低优先级动作删除/后移：优先级降低后需要删除或后移的动作
- **每周怎么更新**：具体的更新操作方式
- **谁来更新**：负责执行该更新操作的人员
- **更新频率/节点**：更新的时间频率或触发节点
- **备注**：补充说明

## 核心原则

真正的执行甘特一定是会动的，不是一张固定图片。必须建立周级滚动更新机制，让甘特表保持生命力。
