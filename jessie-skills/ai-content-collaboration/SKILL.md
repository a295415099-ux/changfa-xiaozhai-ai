# AI内容协同流程表 SOP

## 核心理念

> **AI只有进入流程，才会变成企业能力；否则它永远只是个人技巧。**

三个关键认知：
1. AI如果只服务某一个内容岗，它永远只是工具——只是某个人的个人技巧，不是组织能力
2. AI进入协同流程，才会真正成为企业资产——只有进入流程，AI才会在企业里沉淀成可复制能力
3. 真正的关键不是"用不用AI"，而是"AI有没有进入正确流程位置"——用得多不代表有价值，进入流程才有经营价值

## 适用对象

品牌老板、商家操盘手、内容团队、电商团队、直播团队、运营团队、投放团队、培训讲师。

## 核心目标

解决以下问题：
- "AI只停留在个人工具层"
- "谁会用谁多用一点，但无法沉淀组织能力"
- "内容需求、表达标准、审核、复盘始终没有被AI接入流程"

## 完整SOP流程（17步）

执行本技能时，按以下17个阶段逐步推进。完整细节和表格模板见 `references/full-sop.md`。

### 阶段0：明确本次任务
- **目标**：统一认知——不是再讲AI功能，而是让AI进入协同流程
- **开场话术**：AI如果只服务某一个内容岗，它永远只是工具；AI进入协同流程，才会真正成为资产
- **输出**：AI协同流程任务定义

### 阶段1：回看当前AI使用方式
- **目标**：判断AI现在是个人工具还是组织能力
- **动作**：盘点团队目前谁在用AI、在哪个环节用、是否形成共享标准
- **输出**：当前AI使用现状表

### 阶段2：梳理当前内容业务流
- **目标**：先把现在的流程拉出来
- **动作**：梳理从内容需求提出，到内容上线、复盘的完整路径
- **输出**：当前内容业务流表

### 阶段3：判断内容需求从哪里来
- **目标**：先确定流程起点
- **动作**：明确内容需求来自老板经营目标、品牌方向、电商搜索与承接需求，还是大促节奏需求
- **输出**：内容需求来源表

### 阶段4：需求进入流程（第一步）
- **目标**：让内容需求进入标准化入口
- **动作**：明确谁负责提出内容需求，谁负责把需求转成清晰任务
- **输出**：需求进入流程表

### 阶段5：统一语言系统（第二步）
- **目标**：建立整个流程的统一表达规则
- **动作**：明确谁维护搜索词/卖点词/场景词，谁维护内容表达标准
- **输出**：统一语言系统表

### 阶段6：明确语言系统责任人
- **目标**：让"统一表达"不只是口号
- **动作**：明确三类词由谁来维护，品牌表达由谁来拍板
- **输出**：语言系统责任表

### 阶段7：AI辅助生成（第三步）
- **目标**：明确AI在内容生产环节怎么进入
- **动作**：判断哪些环节可以交给AI做初稿，哪些必须人工判断，如何做多版本适配
- **输出**：AI辅助生成流程表

### 阶段8：判断AI适用边界
- **目标**：给AI使用建立边界
- **动作**：识别哪些内容适合AI辅助生成，哪些必须人工深加工
- **输出**：AI适用边界表

### 阶段9：人工审核（第四步）
- **目标**：明确AI产出后谁来把关
- **动作**：让流程中有明确审核节点：品牌表达标准、平台要求、承接目标
- **输出**：人工审核流程表

### 阶段10：明确审核责任人
- **目标**：保证流程不会卡在"没人拍板"
- **动作**：明确谁审核品牌表达、谁审核平台适配、谁审核承接动作是否到位
- **输出**：审核责任表

### 阶段11：上线与复盘（第五步）
- **目标**：让AI流程不是做到上线就结束
- **动作**：明确内容发出去后谁来看搜索回流、谁来复盘、谁来决定下一轮优化
- **输出**：上线与复盘流程表

### 阶段12：判断AI插入点
- **目标**：把AI准确安放进业务流
- **动作**：判断AI更适合进入需求整理、语言统一、内容生成、平台适配、复盘总结中的哪些环节
- **输出**：AI插入点判断表

### 阶段13：预判流程风险点
- **目标**：提前预判AI流程落地的风险
- **动作**：判断最容易断的地方：需求不清、语言不统一、审核缺失、还是复盘没人做
- **输出**：协同流程风险点表

### 阶段14：形成协同流程草图
- **目标**：把AI进入业务流的路径真正画出来
- **动作**：把"需求→统一语言→AI辅助生成→人工审核→上线与复盘"整合成流程图或流程表
- **输出**：AI内容协同流程表

### 阶段15：输出角色分工
- **目标**：让团队知道谁在流程里负责什么
- **动作**：拆出需求方、规则维护方、AI使用方、审核方、复盘方
- **输出**：协同角色分工表

### 阶段16：输出首轮推进动作
- **目标**：让企业今晚就能先推进一版
- **动作**：给出今晚最值得先推进的1版协同流程动作
- **输出**：首轮推进动作表

### 阶段17：建立后续迭代机制
- **目标**：让协同流程后续可复制、可优化
- **动作**：设定后续如何根据执行情况继续优化流程
- **输出**：协同流程迭代机制表

## 简版逻辑（6步快速版）

当时间有限时，使用简版逻辑：
1. **先看当前流程** → 盘点AI使用方式和内容流程 → 输出当前流程表
2. **明确需求入口** → 明确老板、品牌、电商、节奏四类需求来源 → 输出需求来源表
3. **统一语言系统** → 明确三类词和表达标准由谁维护 → 输出语言系统表
4. **设计AI进入点** → 明确AI负责初稿、版本生成、适配等位置 → 输出AI插入点表
5. **设计审核与复盘** → 明确审核节点和复盘节点 → 输出审核复盘表
6. **串成协同流程草图** → 形成"AI进入内容业务流"的流程草图 → 输出AI内容协同流程表

## 五阶段标准流程（课堂讲解版）

| 流程节点 | 回答的问题 | 课堂解释 |
|---------|-----------|---------|
| 第一步：内容需求从哪里来 | 内容为什么要做、由谁提出 | 可来自老板经营目标、品牌内容需求、电商搜索与承接需求、大促节奏需求 |
| 第二步：统一语言系统 | 内容到底按什么标准说 | 谁维护搜索词/卖点词/场景词，谁维护内容表达标准 |
| 第三步：AI辅助生成 | 哪些环节适合AI先做 | 哪些环节可以交给AI做初稿，哪些必须人工深加工 |
| 第四步：人工审核 | 谁来保证内容不跑偏 | 是否符合品牌表达标准、是否符合平台要求、是否符合承接目标 |
| 第五步：上线与复盘 | 内容上线后谁来负责后续判断 | 谁看搜索回流、谁复盘有效和无效、谁决定下一轮优化 |

## 执行指南

### 基本流程
1. 确认品牌名、团队分工、当前内容流程、课程/项目主题
2. 逐阶段执行SOP的17步（或简版6步），每步输出对应表格
3. 将各阶段结果汇总，形成最终的AI内容协同流程表
4. 输出协同角色分工表和首轮推进动作表
5. 建立迭代机制，确保流程可复制可优化

### 关键原则
- **先看现状再设计**：不先看清现状，就很难把AI插入到正确位置
- **需求来源必须清**：需求来源不清，后面所有动作都会乱
- **语言系统是前提**：这是AI能不能真正进入业务流的关键前提
- **必须有责任人**：没有责任人，统一语言很快就会再次失控
- **AI不是全包**：AI是进入适合它的位置，不是替代所有环节
- **边界必须清晰**：边界不清，团队要么过度依赖AI，要么完全不敢用
- **审核不能省**：不能只要快，还要保证统一、准确、能承接
- **复盘必须放进去**：真正的协同流程，一定要把复盘放进去
- **不追求一口气搭完**：今晚不追求一口气搭完整流程，但必须推进一版

### 五个流程问题（设计时必问）
1. 谁负责提出内容需求？是老板拍板？是品牌负责人拉方向？还是运营和电商提出问题？
2. 谁负责统一词和表达标准？三类词由谁来管？品牌表达由谁来拍板？
3. 谁负责用AI辅助出内容版本？哪些内容适合AI辅助生成，哪些内容必须人工深加工？
4. 谁负责审核是否符合品牌表达标准？不能只要快，还要保证统一
5. 谁负责把内容接回搜索和承接页？不然内容还是会断在中途
