# AI内容表达标准模板 SOP

## 核心认知

本技能解决品牌内容执行层的三个核心问题：
1. **前一张偏品牌层，这一张偏执行层** — 前一张解决"品牌整体怎么说"，这一张解决"团队具体怎么写"
2. **标准模板的价值不是限制创意，而是保证一致性** — 团队在不同人、不同平台、不同时间写内容时，至少有一套统一骨架
3. **真正好的表达模板，不是写得花，而是写得顺** — 开头抓人、卖点自然、种词不硬、结尾有动作，整条链路才顺

## 使用前准备

执行本SOP前，需确认用户已准备好以下输入：
- 品牌表达标准（品牌调性、卖点、场景语言、关键词表达）
- SKU清单（主推款、活动款）

如果用户未提供，先引导用户提供品牌表达标准和核心SKU信息。

## SOP执行流程（17步）

按以下步骤顺序执行，每步完成后输出对应的结果表。

### 步骤0：明确本次任务

**目标**：统一认知——这一张不是品牌表达方向表，而是具体内容怎么写的执行模板。

**动作**：
- 向用户明确：本次任务是围绕核心SKU，产出1个团队可以直接套用的标准内容表达结构
- 强调这是执行层模板，解决的是"具体内容到底怎么写"

**输出**：内容表达模板任务定义

**开场话术**（可选使用）：
> 这一张更偏执行层，它回答的是：团队在日常做具体内容时，到底应该怎么写、怎么展开、怎么保持一致。

### 步骤1：回收品牌表达标准

**目标**：先把前面品牌层的表达规则接过来。

**动作**：
- 汇总前面已经明确的品牌调性、卖点表达方式、场景语言、关键词表达
- 确认：这张表不是另起炉灶，而是承接前面的品牌表达规则

**输入**：品牌表达标准模板、词表、卖点表、场景表

**输出**：`品牌表达基础表`

**输出格式**：
```

| 维度 | 已有规则/标准 |
|------|-------------|

```

**目标**：明确这次先围绕哪个产品写表达模板。

**动作**：

- 不要一上来想给所有产品都做模板
- 先做一个最重要的

### 步骤3：判断内容开头怎么起

**目标**：看哪种开头方式更适合这个SKU。

**动作**：
让AI判断：适合先讲场景、先讲问题、先讲冲突，还是先讲结果。

**判断维度**：
- SKU卖点属性
- 用户痛点类型
- 使用场景特征
- 目标平台属性

**开头方式参考**（详见 [开头结构参考](references/opening-structures.md)）：
- **先讲场景**：适合"使用场景很具体"的SKU
- **先讲问题**：适合"解决某类明确痛点"的SKU
- **先讲冲突**：适合"颠覆原有认知"的SKU
- **先讲结果**：适合"效果可量化"的SKU

**输出**：`开头结构判断表`

**输出格式**：
```
## 开头结构判断表

| 判断维度 | 分析 |
|---------|------|
| SKU特征 | [填写] |
| 用户痛点特征 | [填写] |
| 场景特征 | [填写] |
| 平台属性 | [填写] |
| 推荐开头方式 | [场景/问题/冲突/结果] |
| 推荐理由 | [填写] |
| 备选开头方式 | [填写] |
```

### 步骤5：判断卖点怎么落进内容

**目标**：看卖点应该怎样自然展开。

**动作**：
让AI判断卖点更适合直接讲、通过场景带出，还是通过对比体现。

**三种卖点展开方式**：
- **直接讲卖点**：适合"卖点本身足够强、可感知"的情况
- **通过场景带出卖点**：适合"需要在具体情境中才能体现价值"的情况
- **通过对比体现卖点**：适合"和竞品有明确差异"的情况

**输出**：`卖点展开判断表`

**输出格式**：

### 步骤7：判断搜索词怎么自然进入

**目标**：看关键词怎么进入内容而不显得生硬。

**动作**：
让AI判断品牌词、品类词、卖点词、场景词怎么在内容里自然种下。
**核心原则**：不是硬塞词，而是让用户自然知道下一步该怎么搜。

**四类关键词**（详见 [关键词植入参考](references/keyword-insertion.md)）：
- **品牌词**：让用户记住你的品牌名
- **品类词**：让用户知道搜什么品类
- **卖点词**：让用户带着明确需求来搜
- **场景词**：让用户在相关场景联想时搜到你

**输出**：`搜索词植入判断表`

**输出格式**：
```
## 搜索词植入判断表

| 词类型 | 核心词 | 植入位置 | 植入方式 | 自然度检查 |
|--------|--------|---------|---------|-----------|
| 品牌词 | [填写] | [开头/中间/结尾] | [自然融入/重复强调/场景关联] | [是否自然] |
| 品类词 | [填写] | [位置] | [方式] | [是否自然] |
| 卖点词 | [填写] | [位置] | [方式] | [是否自然] |
| 场景词 | [填写] | [位置] | [方式] | [是否自然] |
```

### 步骤9：判断结尾怎么导向下一步动作

**目标**：看内容结尾应该把人带向哪里。

**动作**：
让AI判断这条内容更适合导搜索、导店、导商品、导直播还是导私域。

**五种结尾导向**：
- **导搜索**：适合"需要用户主动搜才能承接"的内容
- **导店**：适合"店铺有明确承接活动"的内容
- **导商品**：适合"单一爆品驱动"的内容
- **导直播**：适合"直播是主要成交场"的内容
- **导私域**：适合"需要长期关系维护"的内容

**输出**：`结尾导向判断表`

**输出格式**：
```
## 结尾导向判断表

| 判断维度 | 分析 |
|---------|------|
| 内容目标 | [种草/转化/回流/沉淀] |
| 承接路径 | [搜索→店铺/搜索→商品/直播/私域] |
| 当前阶段目标 | [618蓄水/日常种草/新品推广/大促冲刺] |
| 推荐导向方式 | [导搜索/导店/导商品/导直播/导私域] |
```

### 步骤17：建立后续迭代机制

**目标**：让模板后续越用越顺。

**动作**：

## 执行注意事项

1. **开场强调**：本表是执行层模板，解决"具体内容到底怎么写"
2. **先做一个SKU**：不要一上来想给所有产品做模板
3. **四段结构要连贯**：开头→卖点→种词→结尾必须是一体的
4. **种词要自然**：不是硬塞词，而是让用户自然知道该搜什么
5. **结尾要有动作**：真正有效的结尾能推动用户做下一步动作
6. **平台可微调**：标准模板+平台微调 = "统一骨架+平台适配"
7. **分配团队**：不拆到团队，模板就会变成"知道但不用"
8. **建立迭代**：标准模板不是固定不变，而是可迭代的执行资产
