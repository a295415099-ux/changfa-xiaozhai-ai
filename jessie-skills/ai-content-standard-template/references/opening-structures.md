# 开头结构参考

## 四种开头方式详解

### 1. 先讲场景（场景导入型）
**适用条件**：SKU的使用场景具体、可画面化、目标用户容易代入

**公式**：场景画面 → 引发共鸣 → 自然引出SKU

**示例结构**：
- "每次[场景描述]，你是不是也[困扰/感受]..."
- "[时间/地点/情境]下，[人物]正在[做什么]..."
- "有没有人和我一样，[场景共鸣]..."

**优势**：容易让用户停下来，因为看到自己
**风险**：场景不精准会导致用户觉得"这不是我"

### 2. 先讲问题（痛点切入型）
**适用条件**：SKU解决某类明确且高频的痛点

**公式**：抛出一个问题 → 放大痛苦 → 给出解决方案（SKU）

**示例结构**：
- "你是不是也遇到过[具体问题]？"
- "为什么[现象]？核心问题其实是[本质]"
- "[数字/数据]，这件事比你想的更严重"

**优势**：直击需求，转化效率高
**风险**：问题不够痛，用户不会往下看

### 3. 先讲冲突（认知颠覆型）
**适用条件**：SKU颠覆了原有认知或品类惯例

**公式**：原有认知/做法 → 为什么这个认知/做法有问题 → 新的做法/SKU

**示例结构**：
- "99%的人以为[常见做法]是对的，但其实..."
- "以前我们[老方法]，直到发现[新发现]"
- "[行业常见做法]真的对吗？我们测试后发现..."

**优势**：好奇心驱动，完播率高
**风险**：认知差异太大可能引起反感

### 4. 先讲结果（效果展示型）
**适用条件**：SKU效果可量化、可视化、有说服力

**公式**：展示结果/效果 → 倒推怎么做到的 → 引向SKU

**示例结构**：
- "用[SKU]之后，[量化结果]"
- "[Before/After对比]"
- "[用户/自己]做到[效果]的秘密是..."

**优势**：信任感强，直接
**风险**：效果不直观或夸大容易被质疑

## 判断矩阵

| 条件 | 推荐开头方式 |
|------|------------|
| 场景具体 + 用户代入感强 | 先讲场景 |
| 痛点明确 + 解决刚需 | 先讲问题 |
| 品类颠覆 + 认知差异 | 先讲冲突 |
| 效果可量化 + 视觉化 | 先讲结果 |
