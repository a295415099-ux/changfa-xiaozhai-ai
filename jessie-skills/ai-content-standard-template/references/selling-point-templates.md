# 卖点表达模板参考

## 卖点展开三种方式

### 方式1：直接讲卖点
**适用场景**：卖点本身足够强、用户可直观感知

**公式**：
[卖点名称] → [卖点具体是什么] → [这个卖点对用户意味着什么好处] → [一句话总结]

**示例结构**：
- "这个[SKU]最大的特点是[卖点]"
- "很多人问我[问题]，答案就是[卖点]"

### 方式2：通过场景带出卖点
**适用场景**：卖点需要在具体情境中才能体现价值

**公式**：
[一个具体使用场景] → [在这个场景下遇到了什么问题] → [SKU的卖点怎么解决了这个问题] → [用户感受到的变化]

**示例结构**：
- "比如[场景]，一般会[常规做法]，但[SKU]不一样，它[卖点+体验]"
- "[场景还原]，你发现[问题]，这时候[SKU的卖点]就体现出来了"

### 方式3：通过对比体现卖点
**适用场景**：和竞品/行业常规做法有明确差异

**公式**：
[行业/竞品通常怎么做] → [问题或局限] → [SKU做了什么不同的事] → [带来的差异和价值]

**示例结构**：
- "传统的做法是[竞品做法]，问题是[局限]。我们对的做法是[差异化做法]，所以[差异化效果]"
- "市面上大部分[品类]都[特征]，但[SKU]是第一个[差异化]的"

## 卖点展开黄金法则

1. **一个内容聚焦1-2个卖点**，不要贪多
2. **每个卖点必须有"用户语言翻译"**，不能只讲参数
3. **先讲最打动人的卖点**，按重要性排序
4. **卖点之间要有逻辑连接**，不能像清单

## 常见问题

- ❌ 只列功能不连感知（"XX材质" → 应该说"穿上去像第二层皮肤"）
- ❌ 卖点太多太散（一条内容讲5个卖点 = 一个都没讲透）
- ❌ 卖点和用户没关系（"我们升级了XX" → 应该说"升级后你感受到的是YY"）
- ❌ 卖点展开后忘了承接下一步
