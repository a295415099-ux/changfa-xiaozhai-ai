# AI信任证据链建设表 SOP

## 核心目标

解决AI推荐时最重要的不是「相关」而是「可信」，而品牌往往缺少可被AI引用的真实评价、第三方证据、品牌背书和服务承诺的问题。

**核心理念：AI不是只推荐流量大的商品，也会推荐更有证据支撑的商品。**

## 流程概览

共18个步骤（步骤0-17），分5个阶段：

| 阶段 | 步骤范围 | 核心任务 |
|------|----------|----------|
| 基础准备 | 0-1 | 任务定义、资料回收 |
| 顾虑与证据盘点 | 2-7 | 识别核心购买顾虑、匹配证据、盘点评价/内容/品牌/服务四类证据 |
| 整合与生成 | 8-11 | 检查完整度、生成AI可引用表达、生成对应表、生成信任语料 |
| 质量验证 | 12-13 | 检查证据真实性、检查推荐可信度 |
| 落地应用 | 14-17 | 形成正式表、输出缺口清单、团队分工、滚动更新 |

## 执行原则

1. **必须按步骤顺序执行**，不可跳步。前一步的输出是后一步的输入。
2. **每一步都生成明确的表格/文档输出**，不可只口头讨论。
3. **顾虑要用用户语言表达**，不要只写品牌视角。
4. **证据匹配要讲逻辑**：不是所有证据都对所有顾虑有效，要讲匹配关系。
5. **AI可引用表达不能过于广告化**，要能被AI稳定复述。
6. **没有底层资料就停留在概念层**——必须回收真实资料再动手。

## 阶段一：基础准备（步骤 0-1）

### 步骤 0：明确本次任务

**任务目标**：统一认知——这一步不是补几条夸奖，而是建设AI可信推荐所需证据链。

**具体动作**：
- 向用户确认品牌名、本次要处理的商品
- 明确最终交付物：《AI信任证据链建设表》《用户顾虑—证据—话术对应表》《AI可引用品牌信任语料》
- 强调核心理念：AI推荐最重要的不只是相关，而是可信

**输出**：信任证据链任务定义（确认品牌、确认交付物范围）

### 步骤 1：回收基础资料

**任务目标**：先把品牌和商品的信任相关资料拉出来。

**具体动作**：
- 请用户提供：评价截图、买家秀、达人测评内容、媒体报道、资质证书、售后承诺、品牌历史、案例成果、榜单荣誉、复购证据
- 汇总整理为信任基础资料清单

**输出**：信任基础资料表

**注意事项**：没有底层资料，后面只能停留在概念层。

## 阶段二：顾虑与证据盘点（步骤 2-7）

### 步骤 2：识别核心购买顾虑

**任务目标**：明确用户最担心什么。

**具体动作**：
- 梳理用户在购买前最常见的顾虑、怀疑和决策障碍
- 信息来源：评论区、客服咨询、退货原因、竞品差评
- 顾虑要写成用户语言，不要只写品牌视角

**输出格式**：
```
核心购买顾虑表：
| 顾虑编号 | 顾虑描述（用户语言） | 顾虑来源 | 严重程度 |
|---------|-------------------|---------|---------|
```

### 步骤 3：匹配对应信任证据

**任务目标**：明确用什么证据打消顾虑。

**具体动作**：
- 为每一种顾虑匹配最合适的证据类型与证据内容
- 证据类型包括：评价证据、内容证据、品牌证据、服务证据

**输出格式**：
```
顾虑—证据匹配表：
| 顾虑 | 匹配证据类型 | 证据要点 | 匹配逻辑 |
|------|------------|---------|---------|
```

**注意事项**：不是所有证据都对所有顾虑有效，要讲匹配关系。

### 步骤 4：盘点评价证据

**任务目标**：提炼用户评价中的真实场景和真实结果。

**具体动作**：
- 从用户评价中提炼：使用场景、使用结果、体验变化、复购反馈
- 评估每条评价证据的真实度和引用价值

**输出格式**：
```
评价证据表：
| 评价来源 | 使用场景 | 使用结果 | 体验变化 | 真实度 | AI引用价值 |
|---------|---------|---------|---------|-------|----------|
```

**注意事项**：评价证据越真实，AI越容易引用。

### 步骤 5：盘点内容证据

**任务目标**：整理达人、测评、买家秀、案例内容。

**具体动作**：
- 把外部内容证据整理成可调用清单
- 包括：达人测评、测评视频、买家秀、案例内容

**输出格式**：
```
内容证据表：
| 内容类型 | 内容来源 | 核心观点 | 可信度评级 | 适用顾虑 |
|---------|---------|---------|----------|---------|
```

**注意事项**：内容证据是第三方可信度的重要组成。

### 步骤 6：盘点品牌证据

**任务目标**：整理品牌历史、专利、资质、奖项、媒体报道。

**具体动作**：
- 把品牌层面的长期信任资产结构化
- 包括：品牌历史、专利证书、奖项、媒体报道、资质材料

**输出格式**：
```
品牌证据表：
| 证据类型 | 具体内容 | 授予/发布方 | 有效期限 | 信任价值 |
|---------|---------|-----------|---------|---------|
```

**注意事项**：品牌证据解决的是「为什么信这个品牌」。

### 步骤 7：盘点服务证据

**任务目标**：整理发货、退换、质保、客服、安装、售后承诺。

**具体动作**：
- 把服务能力沉淀成可直接引用的证明
- 包括：发货政策、退换规则、售后承诺、客服SOP

**输出格式**：
```
服务证据表：
| 服务类型 | 具体承诺 | 保障力度 | 可引用表达 | 适用顾虑 |
|---------|---------|---------|----------|---------|
```

**注意事项**：服务证据常常直接影响最终下单决策。

## 阶段三：整合与生成（步骤 8-11）

### 步骤 8：检查证据链完整性

**任务目标**：判断每个核心顾虑是否都有证据支撑。

**具体动作**：
- 核对顾虑—评价—内容—品牌—服务之间是否形成闭环
- 用「证据链完整度检查表」逐项评估

**输出格式**：
```
证据链完整度检查表：
| 核心顾虑 | 评价证据 | 内容证据 | 品牌证据 | 服务证据 | 完整度评级 |
|---------|---------|---------|---------|---------|----------|
```

**注意事项**：没有形成证据链，就只是证据堆砌。

### 步骤 9：生成AI可引用表达

**任务目标**：把证据整理成AI能引用的标准话术。

**具体动作**：
- 将各类证据转成结构清楚、可直接引用的表达
- 每个表达需要包含：场景/顾虑 + 证据 + 结论

**输出格式**：
```
AI可引用表达表：
| 引用编号 | 对应顾虑 | 可引用表达 | 引用场景 | 引用强度 |
|---------|---------|----------|---------|---------|
```

**注意事项**：可引用表达不能过于广告化，要能被AI稳定复述。

### 步骤 10：生成用户顾虑—证据—话术对应表

**任务目标**：把顾虑与证据、话术做一一映射。

**具体动作**：
- 围绕每个顾虑生成对应证据和推荐话术
- 这是最适合销售、客服、内容团队共用的表

**输出格式**：
```
用户顾虑—证据—话术对应表：
| 用户顾虑 | 对应证据 | 推荐话术 | 适用角色 | 适用渠道 |
|---------|---------|---------|---------|---------|
```

### 步骤 11：生成AI可引用品牌信任语料

**任务目标**：形成品牌可信推荐语料库。

**具体动作**：
- 将品牌、商品、服务的信任证据整理成语料库
- 每条语料可直接作为AI知识输入

**输出格式**：
```
AI可引用品牌信任语料：
| 语料编号 | 语料内容 | 证据来源 | 引用权重 | 适用场景 |
|---------|---------|---------|---------|---------|
```

**注意事项**：这是最适合给AI作为知识输入的资产。

## 阶段四：质量验证（步骤 12-13）

### 步骤 12：检查证据是否真实可用

**任务目标**：判断是否存在虚泛、夸大、不可验证表达。

**具体动作**：
- 筛掉空泛吹捧、无法验证、过时或不稳定的证据表达
- 逐条检查证据的真实性和可验证性

**输出格式**：
```
证据真实性检查表：
| 证据条目 | 可验证性 | 是否夸大 | 是否过时 | 处理建议 |
|---------|---------|---------|---------|---------|
```

**注意事项**：AI可信推荐建立在真实证据上，不是包装语言上。

### 步骤 13：检查证据是否足够支撑推荐

**任务目标**：判断AI能否从这些证据中总结可信推荐理由。

**具体动作**：
- 复查证据是否足够回答「为什么推荐你」「为什么现在可以买」
- 如果回答不了为什么信、为什么买，链路就不完整

**输出格式**：
```
推荐可信度检查表：
| 检查维度 | 当前状态 | 是否满足 | 缺失说明 |
|---------|---------|---------|---------|
| 「为什么推荐你」 | | ✅/❌ | |
| 「为什么现在可以买」 | | ✅/❌ | |
| 「为什么信这些证据」 | | ✅/❌ | |
| 「和同类比为什么选你」 | | ✅/❌ | |
```

## 阶段五：落地应用（步骤 14-17）

### 步骤 14：形成《AI信任证据链建设表》初版

**任务目标**：把所有结果沉淀成正式工具表。

**具体动作**：
- 将顾虑、证据、话术、语料整合成正式版
- 格式统一、字段完整、可直接使用
- 参考模板见 `references/trust_evidence_chain_template.md`

**输出**：AI信任证据链建设表初版（完整整合）

**注意事项**：这是本技能的核心正式输出。

### 步骤 15：输出当前最关键的信任缺口

**任务目标**：帮团队知道先补哪一类证据。

**具体动作**：
- 从完整度检查表（步骤8）中找出最需要优先补齐的1-3类证据缺口
- 优先补最影响推荐可信度的部分

**输出格式**：
```
当前信任缺口清单：
| 缺口类型 | 影响范围 | 紧急程度 | 补齐建议 | 数据来源 |
|---------|---------|---------|---------|---------|
```

### 步骤 16：分配到团队使用

**任务目标**：让证据链进入内容、电商、客服、投放等使用场景。

**具体动作**：
- 明确不同证据与话术分别给内容、电商、客服、投放使用
- 建立表格式分工清单

**输出格式**：
```
信任资产使用分工表：
| 证据/话术类型 | 使用团队 | 使用场景 | 使用方式 | 更新频率 |
|-------------|---------|---------|---------|---------|
```

**注意事项**：不进入团队使用，证据链就只是资料整理。

### 步骤 17：建立滚动更新机制

**任务目标**：让信任资产随着评价、测评、活动持续增长。

**具体动作**：
- 设计后续按新品、评价新增、达人测评新增、服务政策更新来更新信任链
- 明确更新触发条件、频率和责任人

**输出格式**：
```
信任证据链更新机制表：
| 更新触发条件 | 更新内容 | 更新频率 | 责任人 | 关联步骤 |
|-------------|---------|---------|-------|---------|
| 新品上市 | 新增商品信任证据 | 按需 | 商品负责人 | 步骤1-17 |
| 新增好评 | 更新评价证据表 | 每周 | 客服团队 | 步骤4 |
| 新增达人测评 | 更新内容证据表 | 按需 | 内容团队 | 步骤5 |
| 媒体报道 | 更新品牌证据表 | 按需 | 品牌团队 | 步骤6 |
| 售后政策变更 | 更新服务证据表 | 即时 | 客服团队 | 步骤7 |
```

**注意事项**：信任资产必须持续沉淀，不能一次性盘点结束。
