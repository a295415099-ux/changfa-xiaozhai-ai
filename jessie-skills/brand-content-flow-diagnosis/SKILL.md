# 品牌全域内容流量诊断 Skill

## 核心诊断逻辑

很多品牌不是没有做内容，也不是没有投流量，而是内容、搜索、承接、投放、复购、团队之间没有形成一条完整链路。**任何一个环节断掉，内容流量都会变成"看过就走"。**

完整链路：`内容种草 → 搜索回流 → 店铺承接 → 投放放大 → 复购沉淀 → 团队协同`

## 适用对象

品牌老板、品牌负责人、内容负责人、电商负责人、投放负责人、客服负责人、培训讲师。

## 执行流程

> 完整的 SOP 步骤定义见 `references/sop-flow.md`，模板见 `references/diagnosis-template.md`，概念速查见 `references/core-concepts.md`。

严格按照以下 15 个阶段推进。每个阶段完成后再进入下一阶段，不可跳过。

### 阶段 0: 明确本次任务

向用户确认本次诊断目标：系统识别六大断点，输出《品牌内容流量诊断表》初稿及后续修复清单。

**强调开场语**：品牌不是不努力，而是链路没有真正接起来。你做了内容、投了流量，但如果种草没有通到搜索、搜索没有通到店铺、店铺没有接住流量、投放没有放大有效内容、用户买了就走了，结果就会不稳定。

要求用户提供以下基础信息（缺少则追问）：
- 品牌名
- 当前阶段的经营重点（日常/大促/新品上线等）
- 现有内容平台与店铺平台

### 阶段 1: 回收基础资料

引导用户提供以下资料（尽量全面，但不要求一次性给齐）：
- 站外内容链接（短视频、笔记、图文）
- 搜索数据（品牌词、品类词搜索量）
- 商品详情页、店铺首页截图或链接
- 投放素材与投放数据
- 会员/复购机制文档
- 团队分工与客服话术

将收集到的资料整理为「诊断基础资料表」。

### 阶段 2: 诊断断点一「种草弱」

逐一检查用户提供的内容素材，围绕以下维度诊断：

1. **用户痛点匹配**：内容是否说中了目标用户的核心痛点？
2. **真实场景感**：内容是否让用户产生"这说的不就是我吗"的代入感？
3. **购买意愿激发**：看完内容是否让用户产生搜索或了解的冲动？

**输出**：种草弱诊断结果（是否有此断点 → 具体表现 → 可能原因）。

### 阶段 3: 诊断断点二「搜索弱」

检查内容种草与淘内搜索之间的衔接：

1. **关键词一致性**：外部内容使用的品牌词/品类词/卖点词/场景词，与淘宝商品标题、搜索词是否统一？
2. **搜索引导**：内容中是否明确引导用户搜索哪些关键词？
3. **用户可搜性**：用户看完内容后，是否清楚地知道该搜什么？

**输出**：搜索弱诊断结果。

### 阶段 4: 诊断断点三「承接弱」

检查流量进店后的承接情况：

1. **详情页首屏**：是否延续了种草内容中的核心购买理由？
2. **主图/视频**：是否与外部种草内容调性一致？
3. **评价区/问大家**：是否有正面评价支撑种草主张？是否有未处理的负评？
4. **客服话术**：客服是否了解外部种草内容？是否能强化种草理由？
5. **店铺首页**：新客进店是否能快速找到种草对应的商品？

**输出**：承接弱诊断结果。

### 阶段 5: 诊断断点四「投放弱」

检查投放策略与内容策略的协同：

1. **素材筛选标准**：品牌是否有明确的优质素材筛选标准？
2. **共用指标**：内容团队和投放团队是否使用同一套判断指标（搜索回流、成交承接，不只是点击率）？
3. **素材池机制**：是否有持续更新的优质素材池供投放使用？
4. **数据闭环**：投放数据是否回流指导内容优化？

**输出**：投放弱诊断结果。

### 阶段 6: 诊断断点五「复购弱」

检查首单之后的用户经营：

1. **会员体系**：用户购买后是否有会员承接机制？
2. **私域沉淀**：是否有私域（微信/社群/企微）沉淀路径？
3. **复购内容**：是否有针对老客的复购内容（使用场景、消耗提醒、新品推荐）？
4. **CRM 触达**：是否有基于用户行为的二次触达机制？

**输出**：复购弱诊断结果。

### 阶段 7: 诊断断点六「协同弱」

检查跨团队协同：

1. **关键词统一**：内容、电商、投放、客服是否使用统一的关键词体系？
2. **目标统一**：各部门的目标是否指向同一个增长方向？
3. **指标统一**：复盘时各部门是否看同一套指标？
4. **行动统一**：是否有跨部门协同行动计划？
5. **交付统一**：是否有明确的内容-电商-投放-客服交付流转机制？

**输出**：协同弱诊断结果。

### 阶段 8: 梳理完整链路

将六个断点的诊断结果放回完整链路中：

```
内容种草 → 搜索回流 → 店铺承接 → 投放放大 → 复购沉淀 → 团队协同
```

明确标注每一段的健康状态（正常/弱/严重），找出最先断裂的环节。

**输出**：完整链路状态图（用文字或 ASCII 图展示，标注各段状态）。

### 阶段 9: 判断主断点与次断点

基于完整链路分析，帮用户判断：

- **主断点**（1 个）：当前最影响经营结果的环节，修复它效果最大
- **次断点**（2—3 个）：主断点之外的次要卡点

**输出**：六大断点优先级表（按优先级排序 + 理由）。

### 阶段 10: 输出第一轮修复重点

围绕主断点，输出最值得优先修复的 3—5 个具体动作：

- 动作描述（做什么）
- 预期效果（修到什么程度算修复成功）
- 难度评估（低/中/高）
- 建议顺序

**输出**：链路修复重点清单。

### 阶段 11: 形成《品牌内容流量诊断表》初版

汇总前面所有结果，按 `references/diagnosis-template.md` 的模板格式，生成完整的诊断表。

诊断表包含六大断点每个的：
- 典型表现
- 可能原因
- 诊断问题
- 当前情况
- 优先级（主/次/正常）
- 第一轮修复动作
- 备注

### 阶段 12: 输出团队协同动作

按角色拆解修复动作：

- 内容团队先做什么
- 电商/店铺团队先做什么
- 投放团队先做什么
- 客服团队先做什么
- 复购/CRM 团队先做什么

确保每个角色有明确的交付物和完成时间。

**输出**：团队协同动作表。

### 阶段 13: 建立阶段复盘口径

定义后续验证的指标：

| 断点 | 验证指标 |
|------|----------|
| 种草 | 内容互动率、品牌搜索量变化 |
| 搜索 | 品牌词/品类词搜索量、搜索-进店率 |
| 承接 | 跳失率、转化率、收藏加购率 |
| 投放 | 搜索回流率、投放 ROI |
| 复购 | 复购率、会员活跃度、私域触达率 |
| 协同 | 跨部门交付及时率、统一指标执行率 |

**输出**：阶段复盘口径表。

### 阶段 14: 建立滚动更新机制

帮用户设计诊断表的持续更新节奏：

- 日常更新：每月/每季度检查一次
- 节点更新：大促前中后、新品上线
- 触发更新：新内容投放反馈、搜索趋势明显变化

定义更新负责人和更新流程。

## 关键原则

1. **先诊断再修复**：不要跳过诊断直接给方案，没有诊断的方案是盲目的
2. **主次分明**：不是六个问题一起修，先抓最影响结果的一环
3. **从内容出发**：所有断点诊断以「内容」为起点串联全链路
4. **追问到底**：对模糊描述持续追问，直到定位到可执行的断点
5. **保持品牌视角**：始终以品牌经营者的视角思考，不陷入单个渠道的局部优化
6. **诊断不是结束**：诊断是下一轮动作的起点，必须输出可执行的动作表
7. **动态经营底图**：诊断表不是一次性文件，随经营变化持续更新
