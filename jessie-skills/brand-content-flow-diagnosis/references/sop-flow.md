# SOP 主表 - 品牌全域内容流量诊断完整流程

## 阶段 0: 明确本次任务

- **任务目标**: 统一认知：这一步不是泛泛聊问题，而是系统识别六大断点
- **具体动作**: 明确目标：输出《品牌内容流量诊断表》初稿及后续修复清单
- **输入内容示例**: 品牌名、课程主题、现有内容与店铺资料
- **输出结果**: 诊断任务定义
- **注意事项**: 开场要强调：品牌不是不努力，而是链路没有真正接起来

## 阶段 1: 回收基础资料

- **任务目标**: 先把品牌当前链路相关资料拉出来
- **具体动作**: 汇总站外内容、搜索数据、商品页、店铺页、投放素材、复购机制、团队分工、客服问答等资料
- **输入内容示例**: 内容链接、搜索词、详情页、投放数据、客服话术
- **输出结果**: 诊断基础资料表
- **注意事项**: 没有基础资料，诊断会停留在主观感受

## 阶段 2: 诊断断点一「种草弱」

- **任务目标**: 判断内容是否真正让用户产生购买兴趣
- **具体动作**: 检查内容是否击中用户痛点、是否有真实场景、是否让用户产生"这说的不就是我吗"的感觉
- **输入内容示例**: 短视频、笔记、达人内容、用户互动反馈
- **输出结果**: 种草弱诊断表
- **注意事项**: 不要只看播放和曝光，要看有没有真实种草

## 阶段 3: 诊断断点二「搜索弱」

- **任务目标**: 判断内容种草和淘内搜索是否连上
- **具体动作**: 检查外部种草是否强化品牌词、品类词、卖点词、场景词，用户是否知道该搜什么
- **输入内容示例**: 小红书内容、抖音内容、淘宝搜索词、商品标题
- **输出结果**: 搜索弱诊断表
- **注意事项**: 搜索弱往往是内容关键词和淘内搜索词不统一

## 阶段 4: 诊断断点三「承接弱」

- **任务目标**: 判断流量进店后是否能接住
- **具体动作**: 检查详情页首屏、主图、评价、问大家、客服、店铺首页是否延续了内容中的种草理由
- **输入内容示例**: 商品页、店铺页、主图、评价区、客服话术
- **输出结果**: 承接弱诊断表
- **注意事项**: 承接弱会直接带来跳失高、转化低、收藏加购弱

## 阶段 5: 诊断断点四「投放弱」

- **任务目标**: 判断品牌是否知道哪些内容值得被放大
- **具体动作**: 检查是否建立优质素材筛选标准，内容团队和投放团队是否共用判断指标
- **输入内容示例**: 投放素材、播放数据、转化数据、搜索回流数据
- **输出结果**: 投放弱诊断表
- **注意事项**: 投放不能只看点击，还要看搜索回流和成交承接

## 阶段 6: 诊断断点五「复购弱」

- **任务目标**: 判断活动用户是否有后续经营
- **具体动作**: 检查新客购买后是否有会员、私域、二次需求唤醒、复购内容设计等机制
- **输入内容示例**: 会员策略、复购内容、CRM、老客触达资料
- **输出结果**: 复购弱诊断表
- **注意事项**: 很多品牌的问题不是首单，而是活动后用户沉默

## 阶段 7: 诊断断点六「协同弱」

- **任务目标**: 判断团队是否围绕同一个增长目标协同
- **具体动作**: 检查是否统一关键词、统一内容目标、统一复盘指标、统一行动计划、统一交付机制
- **输入内容示例**: 团队分工、会议节奏、复盘表、行动表
- **输出结果**: 协同弱诊断表
- **注意事项**: 协同弱会导致各部门都很忙，但结果不稳定

## 阶段 8: 梳理完整链路

- **任务目标**: 把六大断点放回完整链路里看
- **具体动作**: 围绕「内容种草 → 搜索回流 → 店铺承接 → 投放放大 → 复购沉淀 → 团队协同」检查哪一段最先断
- **输入内容示例**: 六大断点诊断结果、业务现状、阶段目标
- **输出结果**: 完整链路诊断图
- **注意事项**: 这一步帮助团队看清是局部问题还是系统问题

## 阶段 9: 判断主断点与次断点

- **任务目标**: 帮团队抓住当前最关键的问题
- **具体动作**: 从六大断点里找出当前主断点和 2—3 个次断点
- **输入内容示例**: 全部诊断结果、数据反馈、经营目标
- **输出结果**: 六大断点优先级表
- **注意事项**: 不是六个问题一起修，先抓最影响结果的一环

## 阶段 10: 输出第一轮修复重点

- **任务目标**: 把诊断推进到行动前一步
- **具体动作**: 围绕主断点，输出第一轮最值得优先修复的动作
- **输入内容示例**: 优先级表、现有资源、团队情况
- **输出结果**: 链路修复重点清单
- **注意事项**: 诊断不是结束，而是下一轮动作的起点

## 阶段 11: 形成《品牌内容流量诊断表》初版

- **任务目标**: 把结果沉淀成正式工具表
- **具体动作**: 将六大断点、典型表现、原因、诊断问题、主次判断整合成正式版
- **输入内容示例**: 前面所有诊断结果
- **输出结果**: 品牌内容流量诊断表初版
- **注意事项**: 这是本 Skill 的核心正式输出

## 阶段 12: 输出团队协同动作

- **任务目标**: 让诊断结果进入组织执行
- **具体动作**: 明确内容、电商、投放、客服、复购等角色分别要先做什么
- **输入内容示例**: 诊断表、团队职责、业务流程
- **输出结果**: 团队协同动作表
- **注意事项**: 不拆到团队，诊断表很容易停在会议纪要里

## 阶段 13: 建立阶段复盘口径

- **任务目标**: 让后续验证修复是否生效
- **具体动作**: 明确后续看哪些指标验证种草、搜索、承接、投放、复购、协同是否改善
- **输入内容示例**: 搜索数据、转化数据、内容数据、复购数据
- **输出结果**: 阶段复盘口径表
- **注意事项**: 没有复盘口径，后面很难判断问题有没有被真正修复

## 阶段 14: 建立滚动更新机制

- **任务目标**: 让诊断表随着经营变化持续更新
- **具体动作**: 设计后续如何按大促前中后、活动节点、新内容、新投放反馈更新这张表
- **输入内容示例**: 更新节奏、复盘机制、团队节奏
- **输出结果**: 诊断表更新机制表
- **注意事项**: 诊断表不是一次性文件，而是动态经营底图
