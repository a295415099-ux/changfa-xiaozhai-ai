# 高价值人群拆解 Skill

## 核心逻辑

**内容不是写给「所有消费者」的，而是优先写给最容易被触发、最容易被说服、最容易形成成交的人群。**

**高价值人群不是「最大的人群」，而是**：
- 最容易被你种草的人
- 最容易产生需求的人
- 最容易被你说服的人
- 最容易形成成交的人
- 最值得持续经营的人

**优先拆解五类人群**：新客人群 / 功效（需求）人群 / 场景人群 / 高复购人群 / 价格敏感人群。

## 适用对象

品牌老板、品牌负责人、内容负责人、电商负责人、投放负责人、培训讲师。

## 执行流程

> 完整的提示词模板见 `references/prompts.md`，画像表模板见 `references/templates.md`。

严格按照以下 16 个阶段推进。每个阶段完成后再进入下一阶段，不可跳过。每完成一个阶段向用户汇报中间结果，确认后再推进。

---

### 阶段 0：明确本次任务

**目标**：统一认知，明确本 Skill 不是写泛人群标签，而是拆解高价值人群。

**开场语**（必须强调）：
> 很多人群描述到「25-35岁女性、一二线城市、注重生活品质」就停了。但这种泛标签内容团队根本无法用来写内容——不知道写什么痛点、在哪个场景写、用什么理由说服、让用户买哪款产品。
>
> 今天我们把人群从泛标签升级到「高价值人群画像」，让内容团队能直接拿去写内容、做投放、做电商承接。

**执行动作**：
- 确认品牌名、当前主推产品/品类
- 确认本次任务目标：输出《高价值人群画像表》及后续清单
- 确认输出用途：用于内容创作、投放定向、电商承接

---

### 阶段 1：回收基础资料

**目标**：先把品牌现有用户和商品资料拉出来，防止凭空拆解。

向用户收集以下资料（尽量全面，但不强求一次性给齐），参见 `references/templates.md`「资料收集清单」：

| 资料类别 | 具体内容 |
|---------|---------|
| 用户画像 | 现有成交用户画像、年龄/性别/城市/消费力 |
| 成交数据 | 订单数据、复购数据、客单价分布 |
| 用户反馈 | 评论区内容、客服问答高频问题 |
| 商品矩阵 | 商品清单、主推款/引流款/利润款/形象款 |
| 内容表现 | 各平台内容数据、高互动内容、高转化内容 |
| 搜索数据 | 品牌词、品类词、竞品词搜索情况 |

整理为「人群基础资料表」，发给用户确认。

---

### 阶段 2：统一拆解原则

**目标**：明确什么才算「高价值人群」，建立统一判断口径。

围绕五个维度建立判断逻辑：
1. **种草触发度**：这类人看到什么内容最容易被打动？
2. **需求匹配度**：这类人的需求与品牌产品匹配度如何？
3. **说服容易度**：这类人最容易相信什么证据？
4. **成交转化率**：这类人的下单决策路径是什么？阻力在哪里？
5. **长期经营价值**：成交后能否持续复购、推荐、升单？

输出「高价值人群判断口径表」。

---

### 阶段 3：拆解新客人群

**目标**：识别最值得优先种草的新客群体。

**定义**：对品牌和品类还不熟，但对问题或场景已有初步兴趣的潜在用户。

**拆解维度**见 `references/templates.md`「新客人群拆解框架」：
- 用户是谁（年龄/性别/城市/生活方式）
- 核心痛点（他们遇到什么问题但还没找你解决）
- 首次触达场景（在什么情况下会第一次注意到你）
- 入门购买动机（为什么愿意第一次试试你）
- 决策阻力（什么让他们犹豫不下单）
- 内容触发点（什么内容能让他们产生兴趣）
- 对应产品（入门款/体验款）

使用 `references/prompts.md` 中对应提示词模板。

**注意事项**：新客内容重点是降低理解门槛，不要一上来就讲深奥的专业概念。

---

### 阶段 4：拆解功效/需求人群

**目标**：识别有明确问题、正在找解决方案的人群。

**定义**：已经被某个问题困扰一段时间，正在主动搜索解决方案的精准需求用户。

**拆解维度**见 `references/templates.md`「功效人群拆解框架」：
- 用户是谁
- 核心功效/需求诉求
- 问题严重程度与紧急程度
- 正在搜索什么（搜索词、平台搜索行为）
- 在对比什么（竞品对比维度）
- 需要什么证据才能信（测评/成分/数据/案例）
- 内容触发点
- 对应产品（功效型产品）

**注意事项**：这类人群更适合功效转译、对比测评、证据型内容。不要只讲成分不讲效果。

---

### 阶段 5：拆解场景人群

**目标**：识别需求在特定场景中被激发的人群。

**定义**：平时不一定有需求，但在特定场景（通勤、旅行、熬夜、换季、露营、送礼、聚餐等）中被激发出购买欲望的人群。

**拆解维度**见 `references/templates.md`「场景人群拆解框架」：
- 用户是谁
- 触发场景（什么场景下需要你）
- 场景频率（高频/中频/低频场景）
- 场景中的核心痛点
- 场景情绪（焦虑/期待/社交压力/自我奖赏）
- 内容触发点
- 对应产品（场景型产品/礼盒/限定款）

**注意事项**：场景人群的关键是让用户产生代入感——「这不就是我吗」。

---

### 阶段 6：拆解高复购人群

**目标**：识别已经购买过或天然有复购周期的人群。

**定义**：已经完成首次购买、有复购潜力的老客群体，是最值得持续经营的一类人群。

**拆解维度**见 `references/templates.md`「高复购人群拆解框架」：
- 用户是谁
- 首次购买了什么、为什么买
- 复购触发点（用完了/效果很好/品类消耗快）
- 二次需求（升级需求/搭配需求/替旧需求）
- 升单路径（从入门款到进阶款的路径）
- 推荐意愿与推荐场景
- 内容触发点
- 对应产品（进阶款/组合装/会员权益）

**注意事项**：高复购人群是长期经营价值最高的一类，不要只盯着拉新。

---

### 阶段 7：拆解价格敏感人群

**目标**：识别对活动、权益、组合、折扣敏感的人群。

**定义**：有需求但价格是核心决策变量的人群——不是没钱，而是对性价比要求更高。

**拆解维度**见 `references/templates.md`「价格敏感人群拆解框架」：
- 用户是谁
- 对价格敏感的原因（品类认知低/竞品价格更低/消费习惯/观望期）
- 什么情况下愿意下单（大促/组合装/赠品/限时机制）
- 愿意接受的合理价格带
- 内容触发点
- 对应产品（引流款/组合装/活动款）

**注意事项**：价格敏感人群重点不是低价，而是降低决策阻力。用组合装、赠品、限时权益撬动下单。

---

### 阶段 8：补充人群细节

**目标**：把前面拆解的五类人群从标签升级成可用于内容创作的完整画像。

为每类人群补充以下细节，见 `references/templates.md`「画像补充维度」：
- 一句人话描述（让内容团队一看就懂）
- 典型用户故事/场景
- 关键词库（搜索词 / 痛点词 / 场景词 / 情绪词）
- 内容调性与语言风格建议

输出「高价值人群画像表草稿」，发给用户确认和补充。

---

### 阶段 9：判断人群优先级

**目标**：明确当前阶段最该优先种草谁。

围绕以下维度给五类人群标注优先级（高/中/低），见 `references/templates.md`「优先级判断维度」：
- 品类阶段（新品类先打新客+功效，成熟品类先打场景+复购）
- 商品结构（有引流款更该打新客；有复购款更该打老客）
- 内容能力（团队擅长做哪类内容）
- 转化效率（哪类人群从内容到成交路径最短）
- 竞争环境（哪类人群竞争最少）

输出「5类高价值人群优先级表」。

**注意**：不是五类人群一起做，要先有主次。最多明确 2-3 类高优先级人群。

---

### 阶段 10：生成人群对应内容触发点

**目标**：明确不同人群应该被什么内容打动。

为每类人群生成内容触发点，见 `references/templates.md`「内容触发点框架」：
- 内容触发点（用户看到什么会被触发需求）
- 内容方向（种草笔记/测评对比/场景种草/老客回访/活动预告）
- 说服逻辑（凭什么让他信）
- CTA引导（看完内容后引导做什么）

输出「人群对应内容触发点表」。

---

### 阶段 11：生成人群—产品匹配关系

**目标**：明确不同人群该对应什么产品。

将高价值人群与商品矩阵做对应，见 `references/templates.md`「人群产品匹配框架」：
- 新人 → 入门款/体验款
- 功效需求 → 功效款/明星单品
- 场景触发 → 场景款/礼盒/限定款
- 复购 → 进阶款/组合装/会员款
- 价格敏感 → 引流款/活动组合装

输出「人群—产品匹配表」。

---

### 阶段 12：形成《高价值人群画像表》初版

**目标**：把前面所有拆解结果沉淀成正式工具表。

将以下维度整合，见 `references/templates.md`「高价值人群画像表模板」：

| 维度 | 说明 |
|------|------|
| 人群类型 | 新客/功效/场景/高复购/价格敏感 |
| 用户是谁 | 一句人话描述 |
| 核心痛点 | 他们最在意的问题 |
| 购买动机 | 为什么愿意下单 |
| 内容触发点 | 什么内容能打动他们 |
| 对应产品 | 引导购买什么 |
| 优先级 | 高/中/低 |

这是本 Skill 的核心正式输出，必须完整、可执行。

---

### 阶段 13：输出当前最优先种草人群

**目标**：帮团队聚焦当前阶段最值得做的人群。

从画像表中筛出当前最值得优先种草的 1-3 类人群，形成「下阶段优先种草人群清单」。

清单包含：
- 优先人群类型
- 优先原因（结合业务阶段判断）
- 对应这 3 个月内主推产品
- 建议内容打法方向
- 建议投放定向方向

**目的**：防止团队拿到画像表后仍然回到泛人群表达。

---

### 阶段 14：分配到团队使用

**目标**：让人群画像进入内容、电商、投放等实际执行环节。

明确不同人群画像在不同团队的使用方式，见 `references/templates.md`「团队使用分工表」：

| 团队 | 如何使用画像表 |
|------|-------------|
| 内容团队 | 按画像写内容：对谁说→什么痛点→什么场景→什么理由→引导买什么 |
| 电商团队 | 按人群定向做详情页、直播间话术、店铺推荐 |
| 投放团队 | 按人群画像做投放定向与素材方向 |
| 客服团队 | 按人群特点设计接待话术与推荐逻辑 |
| 会员/私域团队 | 按高复购人群画像设计会员分层与回访机制 |

---

### 阶段 15：建立滚动更新机制

**目标**：让人群画像随着经营和内容反馈持续更新，不是一次性作业。

设计更新机制：

| 更新触发条件 | 更新动作 |
|------------|---------|
| 新品上线 | 更新对应人群画像、产品匹配 |
| 大促节点 | 重新评估价格敏感人群特征 |
| 评论区新反馈 | 补充痛点与购买动机 |
| 搜索词变化 | 更新搜索行为与需求描述 |
| 竞品人群变化 | 关注竞品在打哪类人群 |
| 季度复盘 | 系统更新全部画像表 |

输出「人群画像更新机制表」。

## 最终交付物清单

完成全部 16 个阶段后，应输出以下交付物：

1. **《高价值人群画像表》** — 核心正式输出
2. **《5类高价值人群优先级表》** — 明确先做谁
3. **《人群对应内容触发点表》** — 不同人群的内容打法
4. **《人群—产品匹配表》** — 人群与商品对应关系
5. **《下阶段优先种草人群清单》** — 聚焦 1-3 类人群
6. **《团队使用分工表》** — 各团队如何落地使用
7. **《人群画像更新机制表》** — 持续更新规则
