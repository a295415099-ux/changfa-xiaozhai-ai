# 长发小寨 AI 总控台导航与使用说明

## 这页先看

这是长发小寨 AI 总控台的导航页。它不是单独的报告，而是帮助老板、品牌设计部、电商运营、内容团队快速理解：这个总控台里有什么、应该从哪里开始、每类页面怎么用。

如果你第一次打开总控台，建议按下面顺序看。

## 推荐阅读顺序

| 顺序 | 先看什么 | 用途 |
|---:|---|---|
| 01 | 本导航页 | 先理解总控台怎么用 |
| 02 | AI 项目执行 | 看已经登记的具体 AI 项目，例如商详页、首页、版式 Prompt |
| 03 | 竞品分析 | 看行业、竞品、长发小寨定位和 30/60/90 天动作 |
| 04 | 模板库 | 拿来直接填 brief、登记卡、复盘表、Prompt 模板 |
| 05 | SOP / Jessie Skill 库 | 找具体工作流，例如内容、搜索、承接、包装、社媒 |
| 06 | 资料库说明 / 电商资料库 | 查看已经上传的图片、视频、页面稿、项目素材 |

## 页面编号怎么理解

总控台每个页面左侧都会显示固定编号，例如 `01`、`02`、`03`。

这个编号代表当前页面在总控台里的总顺序，方便团队沟通：

- “你先看 01 导航页”
- “我们今天讨论 30 号 AI项目01”
- “把 17 号固定版式换料模板发给设计师”
- “复盘时回到竞品分析那一页”

编号不是优先级，也不是项目编号。项目编号仍然以文档标题为准，例如 `AI项目01`、`AI项目02`。

## 三种常用使用方式

### 方式一：按栏目看

左侧有栏目筛选：

- 总控导航
- 模板库
- 架构与总控
- Jessie Skill库
- AI项目执行
- 竞品分析
- 资料库说明
- 电商资料库

适合不知道搜什么，但知道自己要看哪类内容时使用。

## 关于私密汇报文件

“汇报文件包”属于更适合老板、一号位和内部会议查看的资料，已从公开总控台默认展示中排除。

需要注意：如果 GitHub 仓库本身是公开的，单纯从网页隐藏并不等于真正私密。真正只对你可见，需要把仓库设为私有，或部署到带登录权限的私有站点。当前处理方式是：公开网页不展示汇报文件包，本地项目里仍保留源文件。

### 方式二：直接搜索

搜索框支持输入关键词，例如：

- `AI项目01`
- `AI项目02`
- `商详页`
- `首页`
- `固定版式`
- `竞品分析`
- `SOP`
- `包装`
- `小红书`
- `悟空`

适合团队开会时快速定位某一页。

### 方式三：把页面当执行模板用

很多页面不是只用来阅读，而是可以直接执行：

- Brief 模板：复制后填入本次需求。
- 检查表：上线前逐项勾。
- Prompt 模板：复制给 AI 或设计师。
- 复盘表：上市后填数据。
- 项目索引：看这个项目有哪些交付物。

## 推荐会议使用方式

开会时可以这样用：

1. 先打开本导航页，确认要讨论哪一类内容。
2. 搜索项目名或关键词，例如 `AI项目01`、`固定版式`。
3. 打开对应页面，按页面编号引用。
4. 如果需要改内容，用“复制”或“GitHub”按钮进入源文件修改。
5. 每次新增资料、截图、Prompt、复盘结果，都归档回对应项目页或资料库。

## 注意事项

- 网页内“编辑”会保存到当前浏览器，适合临时改稿和复制，不等于正式改 GitHub 源文件。
- 正式修改源文件，优先点右上角 `GitHub`。
- 图片、视频、页面稿、包装稿等素材，建议按项目归档到资料库。
- 后续新增 AI 项目时，统一使用 `AI项目03`、`AI项目04` 这样的编号。
- 总控台不是一次性文档，而是长发小寨品牌设计部和 AI 协同的长期工作台。
